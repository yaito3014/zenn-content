---
title: "スマートポインタ"
---

基本編の new と delete の章で、`new` で作った動的なオブジェクトは、`delete` で自分で壊す必要があった。`delete` を忘れればオブジェクトは残り続け、二度 `delete` すれば壊したものをまた壊すことになる。確保と破棄を対にして書く責任が、書く側にあった。基本編は、その責任を引き受ける道具を続きの編で扱うとした。それが **スマートポインタ** である。本章では、動的なオブジェクトをただ一つの持ち主が所有する `std::unique_ptr` と、複数の持ち主で分け合う `std::shared_ptr` を扱う。使うには `<memory>` を取り込む。

## 動的なオブジェクトを所有する

`std::unique_ptr<T>` は、動的に確保した `T` のオブジェクトを所有する。`std::make_unique<T>(...)` で、オブジェクトを確保して、それを所有する `std::unique_ptr` を作る。

```cpp
std::unique_ptr<int> p = std::make_unique<int>(42);
std::println("{}", *p);   // 42
```

`*p` で、所有するオブジェクトを取り出す。`std::unique_ptr` は、自身の寿命が尽きるときに、所有するオブジェクトを `delete` する。言語編のオブジェクトの寿命の章で見た RAII の考え方で、破棄を寿命に結びつける。`delete` を自分で書かないので、`delete` の忘れや二重の `delete` は起きない。

`std::unique_ptr` は、所有するオブジェクトをただ一つだけ持つ。言語編のコピーとムーブの章で見た、コピーを禁じてムーブだけを許す型である。もしコピーできれば、同じオブジェクトを二つの `std::unique_ptr` が持ち、どちらも `delete` して二重の `delete` になりうる。コピーが禁じられているので、これは起きない。

:::message
unique_ptr をコピーする
```cpp
std::unique_ptr<int> p = std::make_unique<int>(42);
std::unique_ptr<int> q = p;
```
`std::unique_ptr` ではコピーが禁じられているので、`p` から `q` へコピーできない。ビルドに失敗する。
:::

持ち主を移すには、ムーブする。ムーブすると、所有が移り、移し元は何も持たなくなる。

```cpp
std::unique_ptr<int> p = std::make_unique<int>(42);
std::unique_ptr<int> q = std::move(p);   // 所有が q へ移る
```

ムーブのあと、`q` がオブジェクトを所有し、`p` は何も持たない。何も持たない `std::unique_ptr` は、空であり、オブジェクトを指さない。

:::message alert
ムーブしたあとの unique_ptr を取り出す
```cpp
std::unique_ptr<int> p = std::make_unique<int>(42);
std::unique_ptr<int> q = std::move(p);
std::println("{}", *p);
```
`p` はムーブで所有を手放し、空になっている。`*p` は、オブジェクトを指さないのに取り出している。ビルドは通るが、結果は定まらず、実行時に深刻な問題が起こりうる。
:::

## 所有を分け合う

オブジェクトを、一つの持ち主に限らず、複数で分け合いたいこともある。`std::shared_ptr<T>` は、所有を分け合う。`std::make_shared<T>(...)` で作る。

```cpp
std::shared_ptr<int> p = std::make_shared<int>(42);
std::shared_ptr<int> q = p;            // p と q が共に所有する
std::println("{}", *q);                // 42
std::println("{}", p.use_count());     // 2
```

`std::shared_ptr` は、`std::unique_ptr` と違ってコピーできる。コピーすると、持ち主が増える。`std::shared_ptr` は、いくつの持ち主が所有しているかを数えており、`use_count` でその数を返す。`p` と `q` の二つが所有しているので、`2` になる。所有するオブジェクトは、最後の持ち主が寿命を終えたときに `delete` される。

`std::unique_ptr` はただ一つの持ち主が所有し、所有を移すにはムーブした。`std::shared_ptr` は複数の持ち主で所有を分け合い、持ち主を増やすにはコピーする。どちらも、確保したオブジェクトの `delete` を引き受け、書く側が `delete` を書かずに済ませる。

:::details 値として振る舞う間接参照(C++26)
`std::unique_ptr` や `std::shared_ptr` では、ムーブやコピーで指し先のオブジェクトそのものは複製されず、所有が移ったり分かれたりする。これとは別に、コピーすると指し先の値ごと複製する、値のように振る舞う間接参照が、C++26 で `<memory>` に加わる。`std::indirect<T>` は、動的に確保した `T` の値を一つ持つ。`std::polymorphic<T>` は、`T` かそれを継承した型の値を持ち、コピーのときは、その時点の型のまま複製する。どちらも、クラスのメンバとして動的なオブジェクトを、参照ではなく値の意味で持ちたいときに使う。

これらが指し先を持たなくなるのは、ムーブされた後だけである。この章で見た、ムーブされたあとのスマートポインタと同じ事情である。
:::

基本編で生の道具として触れた配列や `new`/`delete` は、ここで標準ライブラリの道具に置き換わった。ここまで見てきたのは、値を並べて持つコンテナ、それを操作するアルゴリズム、範囲を所有せずに参照する `std::span`、複数の値をまとめる `std::pair`・`std::tuple`、そして動的なオブジェクトの所有を引き受けるスマートポインタである。いずれも、値をどう持ち、参照し、所有するかを扱う道具だった。次章からは、一つの値が持ちうる形(あるかないか、値かエラーか、いくつかの型のうちどれか)を表す型を見ていく。

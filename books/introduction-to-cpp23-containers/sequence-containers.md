---
title: "シーケンスコンテナ"
---

基本編の配列の章で、同じ型の値を並べて持つ生の配列を見た。範囲外アクセスや、大きさの管理を自分で引き受ける難しさに触れ、確保・破棄・大きさの管理を引き受ける標準ライブラリの道具(`std::vector` と `std::array`)を続きの編で扱うとした。本章では、その2つに加えて、両端に足せる `std::deque` も扱う。値を巡るための標準のイテレータと、要素を指していたものが指す先を失う無効化も、ここで見る。

## 大きさを変えられる並び

`std::vector<T>` は、`T` の値を並べて持つコンテナである。後から要素を足して、大きさを変えられる。使うには `<vector>` を取り込む。

```cpp
std::vector<int> v = {1, 2, 3};
v.push_back(4);
for (int x : v) std::println("{}", x);   // 1 2 3 4
std::println("{}", v.size());            // 4
```

`std::vector<int>` の `<int>` は、言語編のテンプレートの章で見たとおり、クラステンプレート `vector` に型 `int` を与えたものである。`push_back` は末尾に要素を足す。`size()` は持っている要素の数を返す。生の配列と違い、`std::vector` は要素を格納する領域を自分で引き受けるので、`new[]`/`delete[]` を自分で対にして書く必要はない。

## 大きさの決まった並び

`std::array<T, N>` は、`T` の値を `N` 個並べて持つコンテナである。`N` はビルドの時点で決まり、後から変えられない。使うには `<array>` を取り込む。

```cpp
std::array<int, 3> a = {1, 2, 3};
for (int x : a) std::println("{}", x);
std::println("{}", a.size());   // 3
```

`std::array` は、大きさが固定という点で生の配列に近いが、`size()` で大きさを問えて、文字列の章で見た、範囲を確かめる `at` も持つ。大きさを変える必要がなければ `std::array`、変えるなら `std::vector` を使う。

## 両端に足せる並び

`std::vector` は末尾に `push_back` で要素を足した。先頭にも末尾にも足したいときは、`std::deque` を使う。使うには `<deque>` を取り込む。

```cpp
std::deque<int> d = {2, 3};
d.push_front(1);
d.push_back(4);
for (int x : d) std::println("{}", x);   // 1 2 3 4
```

`push_front` は先頭に、`push_back` は末尾に要素を足す。`std::vector` が末尾だけだったのに対し、`std::deque` は両端に足せる。添字 `[]` や範囲for は `std::vector` と同じように使える。

## イテレータで巡る

コンテナの要素は、`begin()` と `end()` が返す **イテレータ** で巡れる。言語編のイテレータと範囲forの章で、自分の型に `begin`/`end` と最小のイテレータ・センチネルを実装して範囲for を回した。標準のコンテナは、その標準のイテレータをあらかじめ備えている。

```cpp
std::vector<int> v = {10, 20, 30};
for (auto it = v.begin(); it != v.end(); ++it)
    std::println("{}", *it);   // 10 20 30
```

`begin()` は先頭の要素を指すイテレータ、`end()` は末尾の次を指すイテレータを返す。`*it` で指す先の要素を取り出し、`++it` で次へ進め、`it != v.end()` で終端に達したかを判定する。言語編では `end` はセンチネルを返したが、配列のイテレータのコラムで見たように、終端の印はイテレータが兼ねることもある。標準のコンテナでは、`begin()` と `end()` は同じイテレータ型を返し、`end()` のイテレータがその印を兼ねる。言語編で見たとおり、`for (int x : v)` の範囲for は、この `begin`/`end` とイテレータの操作への脱糖である。

## 範囲の外を読む

文字列の章で見た `[]` と `at` の使い分けは、シーケンスコンテナにもあてはまる。`[]` は添字が範囲の内にあるかを確かめない。

:::message alert
範囲の外を添字で読む
```cpp
std::vector<int> v = {1, 2, 3};
std::println("{}", v[5]);
```
`v` は3要素なので、添字 `5` は範囲の外にある。`[]` は範囲を確かめないので、ビルドは通るが、コンテナの外を読む。結果は定まらず、実行時に深刻な問題が起こりうる。
:::

範囲を確かめてほしいときは `at` を使う。`at` は、添字が範囲の外なら `std::out_of_range` を投げる。

```cpp
std::vector<int> v = {1, 2, 3};
v.at(5);   // std::out_of_range を投げる
```

## 無効化

`std::vector` の大きさを変える操作には、注意するところがある。`std::vector` が要素を格納する領域には限りがあり、`push_back` で要素数がその限りを超えると、より大きな別の領域が確保され、要素はそちらへ移される。元の領域は解放される。このとき、移される前の要素を指していたイテレータやポインタ、参照は、解放された領域を指したままになる。これを **無効化** と呼ぶ。無効化されたイテレータやポインタ、参照を使うことが、誤りになる。

:::message alert
再確保で無効化された参照を使う
```cpp
std::vector<int> v = {1, 2, 3};
int& first = v[0];
v.push_back(4);
std::println("{}", first);
```
`first` は先頭の要素を指す参照である。`push_back` で要素数が確保済みの領域を超えると、要素は別の領域へ移され、元の領域は解放されることがある。`first` は解放された領域を指したままになる。ビルドは通るが、`first` を使うと、結果は定まらず、実行時に深刻な問題が起こりうる。
:::

無効化されたものを使うことは、言語編のオブジェクトの寿命の章で見た、破棄されたオブジェクトを指す参照を使うのと同じ誤りである。再確保では、元の領域にあった要素は破棄され、その領域は解放される。新しい領域には同じ値を持つ別の要素が構築されるが、`first` が指すのは破棄された方であって、新しい要素ではない。同じことは、`push_back` の前に取った `begin()` のイテレータや、要素を指すポインタにもあてはまる。`std::vector` の大きさを変える操作の後は、その前に取ったイテレータやポインタ、参照を使わない。必要なら、操作の後に取り直す。

`std::vector` と `std::array` は、同じ型の値を並べて持つシーケンスコンテナだった。次章では、値そのものでなく、値をキーで引く連想コンテナ(`std::map` と `std::set`)を見ていく。

---
title: "型エイリアス"
---

型には別の名前を付けられる。既存の型に与えた別名を **型エイリアス** と呼ぶ。

## 型に別名を付ける

`using 新しい名前 = 既存の型;` と書く。

```cpp
using Coord = int;
```

これで `Coord` は `int` の別名になる。`Coord` と書いた場所は、`int` と書いたのと同じである。

```cpp
Coord x = 3;
int y = x;
```

`Coord` の `x` を、`int` の `y` にそのまま渡せる。

型エイリアスは、新しい型を作らない。基本編の列挙型やクラスは新しい型を導入したが、型エイリアスは既存の型に別の名前を与えるだけで、`Coord` と `int` は同じ型である。

## 長い名前や繰り返しに名前を与える

別名が効くのは、型の名前が長いときや、同じ型が何度も現れるときである。たとえば、座標を表す `int` に `Coord` という名前を与えておく。

```cpp
using Coord = int;

struct Point
{
    Coord x;
    Coord y;
};
```

座標であることを `Coord` という名前で表せる。座標の型を後で変えたくなったときも、`using Coord = ...;` の1箇所を直せば、`Coord` を使ったすべての場所に反映される。

:::details typedef による書き方
型エイリアスには、`typedef` という書き方もある。`using` より前からあるものである。

```cpp
typedef int Coord;
```

`using Coord = int;` と同じ意味だが、名前と型の順序が逆である。`using` は、左辺が新しい名前・右辺が既存の型という並びで、変数の初期化の `=` と形が揃う。この編では `using` を使う。
:::

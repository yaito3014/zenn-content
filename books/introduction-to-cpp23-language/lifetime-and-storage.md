---
title: "オブジェクトの寿命と記憶域期間"
---

基本編では、宣言した時点で作られてブロックを抜けると破棄されるオブジェクトと、`new` と `delete` で作る時点と壊す時点を自分で決めるオブジェクトとを見た。オブジェクトが作られてから破棄されるまでを、そのオブジェクトの **寿命** と呼ぶ。オブジェクトがいつ作られ、いつ破棄されるかは、その **記憶域期間** によって変わる。本章では、自動・動的・静的の3つの記憶域期間と寿命、そして寿命に資源を結びつける RAII を扱う。記憶域期間にはこのほかに、`thread_local` を付けたオブジェクトが持つ、スレッドごとの記憶域期間もあるが、本章では立ち入らない。

## 自動記憶域期間

ブロックの中で宣言したオブジェクトは、`static` や `thread_local` を付けない限り、**自動記憶域期間** を持つ。宣言した時点で作られ、そのブロックを抜けるときに破棄される。

確認のため、作成と破棄を出力する `Guard` を考える。

```cpp
struct Guard
{
    Guard()  { std::println("作成"); }
    ~Guard() { std::println("破棄"); }
};
```

```cpp
std::println("ブロックの前");
{
    Guard g;
    std::println("ブロックの中");
}
std::println("ブロックの後");
```

出力は次のようになる。

```
ブロックの前
作成
ブロックの中
破棄
ブロックの後
```

`g` は宣言した時点で作られ、ブロックの `}` で破棄される。「ブロックの後」より前に「破棄」が出ている。寿命がブロックに結びついている。

## 動的記憶域期間

`new` で作ったオブジェクトは、**動的記憶域期間** を持つ。ブロックを抜けても破棄されず、`delete` を書いた時点で破棄される。寿命を自分で決める。

```cpp
std::println("new の前");
Guard* p = new Guard();
std::println("delete の前");
delete p;
std::println("delete の後");
```

```
new の前
作成
delete の前
破棄
delete の後
```

`new Guard()` の時点で作られ、`delete p;` の時点で破棄される。基本編の new と delete の章で見たとおり、`delete` を忘れれば破棄されず、リークする。

## 静的記憶域期間

**静的記憶域期間** を持つオブジェクトは、プログラムが終わるまで残りつづける。関数の中で `static` を付けて宣言したオブジェクトは、その一例である。

```cpp
int next_id()
{
    static int id = 0;
    id = id + 1;
    return id;
}
```

`id` は静的記憶域期間を持つ。`0` という定数で初期化されるので、その寿命はプログラムの開始時に始まる。`next_id()` を呼んでも作り直されず、同じ `id` が前の値を保って増える。

```cpp
next_id();   // 1
next_id();   // 2
next_id();   // 3
```

`id` が破棄されるのは、プログラムが終わるときである。

:::details `static` という語
`static` は、文脈によって別のことを表す。static メンバの章では、クラスに1つだけ持つメンバを表した。ODR と inline の章のコラムでは、名前空間スコープでの内部リンケージを表した。ここでの `static` は、関数の中のオブジェクトに静的記憶域期間を持たせる。同じ語が、別の意味で使われる。
:::

## RAII

自動記憶域期間を持つオブジェクトは、ブロックを抜けるときに破棄される。`return` で途中で抜けたときも、例外でブロックを抜けるときも同じである(例外でのふるまいは例外の章で扱う)。クラス型なら、破棄のときにデストラクタが呼ばれる。この破棄に、資源の後始末を結びつけられる。

コピーとムーブの章の `Buffer` は、コンストラクタで配列を `new[]` で確保し、デストラクタで `delete[]` した。`Buffer` をブロックの中で宣言すれば、ブロックを抜けるときにデストラクタが呼ばれ、配列が解放される。`delete[]` を自分で書く必要はない。

```cpp
{
    Buffer b(10);
    // b を使う
}   // ここで b のデストラクタが呼ばれ、配列が解放される
```

資源の確保をコンストラクタに、解放をデストラクタに結びつけ、オブジェクトの寿命に資源の寿命を合わせる。この作法を、英語では RAII(Resource Acquisition Is Initialization)と呼ぶ。`new` と `delete` を手で対にする代わりに、資源を持つオブジェクトの寿命に任せる。

基本編の new と delete の章で橋渡しした `std::unique_ptr` などのスマートポインタは、この RAII で動的なオブジェクトの破棄を引き受ける。それらはこの先の編で扱う。

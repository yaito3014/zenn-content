---
title: "メモリレイアウト"
---

前章では、オブジェクトの記憶域がいつからいつまで続くか、その寿命を見た。記憶域には、もう一つの面がある。オブジェクトがどれだけの大きさで、メモリにどう並ぶかである。本章では、オブジェクトの大きさ、それを左右するアライメント、空のメンバも場所を取ること、そして空のメンバを詰める `[[no_unique_address]]` を扱う。

## オブジェクトの大きさ

オブジェクトは、メモリ上に場所を占める。その大きさは、`sizeof` で得られる。`sizeof(T)` は、型 `T` のオブジェクト1つが占めるバイト数である。

```cpp
struct Point
{
    int x;
    int y;
};
```

```cpp
std::println("{}", sizeof(Point));   // 8
```

`int` が4バイトの環境では、`Point` は8バイトになる(`sizeof` の値は環境による)。ここで見るような単純なクラスでは、データメンバは宣言した順にメモリへ並ぶ。ただし、クラスの大きさは、データメンバの大きさの単純な合計になるとは限らない。

## アライメントとパディング

型にはそれぞれ **アライメント** がある。これは、その型のオブジェクトが置かれるアドレスの境界で、`alignof` で得られる。多くの環境で `alignof(int)` は4で、`int` は4の倍数のアドレスに置かれる。

データメンバのアライメントをそろえるため、メンバの間に **パディング**(詰め物)が入ることがある。

```cpp
struct Pad
{
    char c;
    int n;
};
```

```cpp
std::println("{}", sizeof(Pad));   // 8
```

`char` は1バイト、`int` は4バイトなので、単純に足せば5バイトである。だが `n` を4の倍数のアドレスにそろえるため、`c` の後に3バイトのパディングが入り、`sizeof(Pad)` は8になる。クラス全体の大きさも、アライメントの倍数になるよう、末尾にパディングが入ることがある。データメンバを並べる順序によって、パディングの量、ひいては `sizeof` が変わることもある。

## 空のメンバも場所を取る

データメンバを1つも持たないクラスも書ける。

```cpp
struct Empty {};
```

`Empty` はデータメンバを持たない。それでも、`Empty` のオブジェクトは大きさを持つ。

```cpp
std::println("{}", sizeof(Empty));   // 1
```

`sizeof(Empty)` は `0` ではなく、少なくとも `1` になる。別々のオブジェクトが別々のアドレスを持つよう、空の型にも大きさを持たせる決まりである。

この空のクラスを、別のクラスのデータメンバにする。

```cpp
struct Holder
{
    Empty e;
    int n;
};
```

```cpp
std::println("{}", sizeof(Holder));   // 8
```

データメンバの `e` は、既定では自分のアドレスを持ち、空でも1バイトを占める。そのうえで `n` を4の倍数のアドレスにそろえるパディングが入り、`Holder` は8バイトになる。`int` の `n` だけのときの4バイトより大きい。

## [[no_unique_address]]

空のメンバが取る場所は、`[[no_unique_address]]` を付けると詰められる。

```cpp
struct Packed
{
    [[no_unique_address]] Empty e;
    int n;
};
```

```cpp
std::println("{}", sizeof(Packed));   // 4
```

`[[no_unique_address]]` を付けた `e` は、ほかのメンバと場所を分け合ってよい。多くの環境では `e` が場所を取らなくなり、`Packed` は `int` の `n` だけの4バイトになる。ただし、これは実装が場所を詰めてよいという許可であって、必ず詰められるとは限らない。

`[[no_unique_address]]` は、属性の章で見た `[[...]]` の形の属性の一つである。属性の章で、これを取り除くとオブジェクトが大きくなりうると触れたのは、この働きによる。

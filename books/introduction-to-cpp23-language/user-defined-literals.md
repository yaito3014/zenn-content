---
title: "ユーザー定義リテラル"
---

基本編から、`42` や `3.14`、`"hello"` のようなリテラルを書いてきた。`42u` の `u` や `3.14f` の `f` のように、リテラルにはサフィックスを付けられるものがある。サフィックスを付けたリテラルから、自分の型の値を作ることもできる。これを **ユーザー定義リテラル** と呼ぶ。本章では、サフィックスを定義するリテラル演算子、サフィックスの名前の規約、そしてリテラルの種類によって受け取る型が決まることを扱う。

## サフィックスを定義する

例として、長さをメートルで表す `Meters` を、`1.5_m` のように書けるようにする。サフィックス `_m` を、`operator""` に続けて名前を書いた関数として定義する。

```cpp
struct Meters
{
    long double value;
};

Meters operator""_m(long double v)
{
    return {.value = v};
}
```

`operator""` に続けてサフィックスの名前を書いたこの関数を **リテラル演算子** と呼ぶ。演算子オーバーロードの章で `+` を `operator+` という関数として定義したように、サフィックス `_m` を `operator""_m` という関数として定義する。これで `1.5_m` と書ける。

```cpp
Meters len = 1.5_m;
std::println("{}", len.value);   // 1.5
```

この定義のもとで、`1.5_m` は `operator""_m(1.5L)` の呼び出しとして扱われる。サフィックスを除いた数値が `long double` の値として引数 `v` に渡り、その値を持つ `Meters` が返る。

## サフィックスは `_` で始める

サフィックスの名前には決まりがある。`_` で始まらないサフィックスは、規格が将来の標準化のために予約している。自分で定義するサフィックスは、`_` で始める。`_m` の先頭の `_` はこのためである。

`_` で始まらないサフィックスは予約されているので、それを使うのは標準ライブラリが提供するリテラルに限られる。文字列を `std::string` にする `s` や、時間を表す `min`・`h` などがそれにあたる。これらはこの先の編で扱う。

## リテラルの種類と受け取る型

これまで定義したリテラル演算子は、リテラルの数値をそのまま引数に受け取る。この形では、引数の型はリテラルの種類で決まる。`1.5_m` のように小数のリテラルに付けるなら、引数は `long double` になる。整数のリテラルに付けるなら、引数は `unsigned long long` になる。数値をその文字の並びのまま受け取る形のリテラル演算子もあるが、本章では立ち入らない。

例として、バイト数を表す `Bytes` を、整数のリテラルから作る。

```cpp
struct Bytes
{
    unsigned long long value;
};

Bytes operator""_B(unsigned long long v)
{
    return {.value = v};
}
```

```cpp
Bytes b = 512_B;
std::println("{}", b.value);   // 512
```

この定義のもとで、`512_B` は `operator""_B(512ULL)` の呼び出しとして扱われる。整数なので、`unsigned long long` の値として引数に渡る。小数の `_m` と整数の `_B` とで、受け取る型が違う。

ここで見た形のリテラル演算子は、受け取る型がリテラルの種類に対応している。そのため、整数用の `_B` だけを定義しても、小数の `512.0_B` には使えない。逆に、小数用の `_m` だけを定義しても、整数の `1_m` には使えない。

:::message
種類の合わないリテラルにサフィックスを付ける
```cpp
Meters len = 1_m;
```
`_m` は小数のリテラルを受け取る `long double` 版しか定義していない。整数の `1` に付けると、対応するリテラル演算子が見つからず、ビルドに失敗する。
:::

:::details 文字列のリテラル
文字列のリテラルにもサフィックスを付けられる。この場合、文字列の先頭を指すポインタと、その長さの二つを引数に取るリテラル演算子を定義する。

```cpp
#include <cstddef>

struct Tag
{
    char const* text;
};

Tag operator""_tag(char const* s, std::size_t)
{
    return {.text = s};
}
```
この定義のもとで、`"hello"_tag` は `operator""_tag("hello", 5)` の呼び出しとして扱われる。一つ目の引数に文字列の先頭、二つ目の引数にその長さ `5` が渡る(この例では長さは使っていない)。
:::

---
title: "可変長テンプレートと特殊化"
---

テンプレートの章では、型をパラメータにとるテンプレートを書いた。本章では、その発展として、引数をいくつでも受け取る **可変長テンプレート** と、特定の型に別の定義を与える **特殊化** を扱う。

## 可変長テンプレート

基本編の出力の章で使った `std::println` は、`std::println("{} {}", a, b)` のように引数をいくつでも受け取れる。引数の個数を固定せず、いくつでも受け取れるテンプレートを **可変長テンプレート** と呼ぶ。

例として、受け取った値をすべて足す `sum` を書く。

```cpp
template <typename... Ts>
auto sum(Ts... args)
{
    return (args + ...);
}
```

`typename... Ts` は、0個以上の型をまとめて表す。これを **パラメータパック** と呼ぶ。`Ts... args` は、その型の引数をまとめて受け取る。戻り値の型は `auto` とし、`return` する式から決めている(基本編で見た `auto`)。

`(args + ...)` は、パックの各引数を `+` でつないだ式に展開される。`sum(1, 2, 3)` なら `1 + 2 + 3` になる。パックを一つの演算子でつなぐこの式を **畳み込み式**(英語では fold expression)と呼ぶ。

```cpp
sum(1, 2, 3);     // 1 + 2 + 3 で 6
sum(1.5, 2.0);    // 1.5 + 2.0 で 3.5
```

`sum(1, 2, 3)` は `6` を、`sum(1.5, 2.0)` は `3.5` を返す。引数の個数も型も、呼び出しごとに変わってよい。`std::println` が引数をいくつでも受け取れるのも、同じく引数をパックで受け取っているからである。

ただし、畳み込み式 `(args + ...)` 自体は、引数が1つ以上あって初めて成り立つ。引数が0個だと、`+` でつなぐものがない。

:::message
引数なしで `sum` を呼ぶ
```cpp
sum();
```
`sum()` は `args` が0個のパックで実体化される。`(args + ...)` は `+` でつなぐものがなく、式が定まらない。ビルドに失敗する。
:::

空のパックでも成り立たせるには、足し合わせの初項を与える。`(0 + ... + args)` は、初項 `0` から各引数を `+` でつなぐ畳み込み式である。

```cpp
template <typename... Ts>
auto sum(Ts... args)
{
    return (0 + ... + args);
}
```

```cpp
sum();            // 0
sum(1, 2, 3);     // 0 + 1 + 2 + 3 で 6
sum(1.5, 2.0);    // 0 + 1.5 + 2.0 で 3.5
```

引数が0個なら、結果は初項の `0` になる。`sum()` は `0` を返し、ビルドに成功する。引数があるときは初項の `0` を足すだけなので、結果は変わらない。

:::details 畳み込みの向き
畳み込み式には、つなぐ向きがある。`(args + ...)` は右から、`(0 + ... + args)` は左からつなぐ。`sum(1, 2, 3)` なら、前者は `1 + (2 + 3)`、後者は `((0 + 1) + 2) + 3` になる。`+` ではどちらでも結果は同じだが、`-` のように向きで結果が変わる演算子では、つなぐ向きが効いてくる。
:::

## 特殊化

テンプレートは、すべての型に同じ定義を与える。だが、特定の型にだけ別の定義を与えたいことがある。これを **特殊化** と呼ぶ。

型の名前を文字列で返す `TypeName` を考える。まず、どんな型に対しても `"unknown"` を返す定義を書く。

```cpp
template <typename T>
struct TypeName
{
    static char const* get() { return "unknown"; }
};
```

`int` に対しては `"int"` を返したい。`int` 用の定義を別に書く。

```cpp
template <>
struct TypeName<int>
{
    static char const* get() { return "int"; }
};
```

`template <>` で始め、`TypeName<int>` と型を名指しする。これが `TypeName` の `int` に対する特殊化である。

```cpp
TypeName<int>::get();      // "int"
TypeName<double>::get();   // "unknown"
```

`TypeName<int>::get()` は `int` 用の定義が使われ `"int"` を返す。`TypeName<double>::get()` は名指しした定義がないので、もとの定義が使われ `"unknown"` を返す。

## 部分特殊化

`int` のように型を一つ名指しするのではなく、「ポインタ型すべて」のように、型の一群に別の定義を与えることもできる。これを **部分特殊化** と呼ぶ。ポインタ型に対する `TypeName` を書く。

```cpp
template <typename T>
struct TypeName<T*>
{
    static char const* get() { return "pointer"; }
};
```

`TypeName<T*>` は、`T` が何であれ、ポインタ型 `T*` に対する定義である。`T` の部分は、定義の中で型パラメータとして使える。

```cpp
TypeName<int*>::get();      // "pointer"
TypeName<double*>::get();   // "pointer"
```

`int*` も `double*` もポインタ型なので、こちらの定義が使われ、どちらも `"pointer"` を返す。

部分特殊化は、クラステンプレートに対するものである。関数テンプレートには部分特殊化はなく、同じ名前で引数の違う関数を複数定義する(基本編の型と関数の章で見たオーバーロード)ことで、型ごとの振り分けを行う。

---
title: "再帰"
---

関数は、自分自身を呼び出せる。これを **再帰** と呼ぶ。本章では、再帰の書き方、再帰での呼び出しの積み重なり、そして止まらない再帰の問題を扱う。

## 再帰

階乗を再帰で書く。`n` の階乗は、`n` から `1` までを掛けた値で、`0` の階乗は `1` と定める。

```cpp
int factorial(int n)
{
    if (n == 0) return 1;
    return n * factorial(n - 1);
}
```

`factorial(n)` は、`n` が `0` なら `1` を返し、そうでなければ `n` に `factorial(n - 1)` を掛けて返す。`factorial` の本体から `factorial` を呼んでいる。これが再帰である。

```cpp
std::println("{}", factorial(5));   // 120
```

`factorial(5)` は `5 * factorial(4)`、`factorial(4)` は `4 * factorial(3)`、と続き、`factorial(0)` が `1` を返す。これらを掛け合わせて、`5 * 4 * 3 * 2 * 1 * 1` で `120` になる。

`n == 0` のときに再帰せず `1` を返す部分を **停止条件** と呼ぶ。停止条件は、再帰を止めるためにいる。これがないと、再帰は止まらない。

## 呼び出しの積み重なり

関数を呼ぶたびに、その呼び出しのための引数や局所変数が作られる。寿命と記憶域期間の章で見たとおり、これらは自動記憶域期間を持ち、呼び出しが終わると破棄される。再帰では、`factorial(5)` が `factorial(4)` を、それが `factorial(3)` を、と、停止条件に達するまで呼び出しが積み重なる。それぞれの呼び出しが、自分の `n` を持つ。

```
factorial(5)
  factorial(4)
    factorial(3)
      factorial(2)
        factorial(1)
          factorial(0)   // 停止条件
```

停止条件の `factorial(0)` が `1` を返すと、積み重なった呼び出しが、内側から順に結果を返して戻っていく。多くの環境では、この呼び出しごとの場所は **スタック** と呼ばれる領域に積まれる。

## 止まらない再帰

停止条件に達しないと、呼び出しは積み重なり続ける。停止条件があっても、そこに達しなければ同じである。`factorial` は `n == 0` で止まるが、負の数を渡すと、`n` は `0` を通り越して減り続け、停止条件に達しない。

:::message alert
停止条件に達しない再帰
```cpp
std::println("{}", factorial(-1));
```
`factorial(-1)` は `factorial(-2)`、`factorial(-3)`、と呼び出しを続け、`n == 0` に達しない。ビルドは通るが、実行すると呼び出しが積み重なり続けてスタックを使い尽くし、実行時に深刻な問題が起こりうる。
:::

停止条件は、すべての呼び出しから必ず達するように書く。`factorial` なら、負の数を受け取らないようにする、あるいは負の数に対する停止条件も用意する、といった備えがいる。

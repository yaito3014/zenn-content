---
title: "= default と = delete"
---

コンストラクタやデストラクタなど、自分で書かなくても、条件に応じて自動で定義される特定のメンバ関数を **特殊メンバ関数** と呼ぶ。`= default` は、その定義を自動生成にまかせると明示する書き方である。`= delete` は、関数を禁じると明示する書き方で、特殊メンバ関数に限らずどの関数にも使える。

## = default

`= default` は、特殊メンバ関数を明示的に宣言したうえで、その定義を自動生成にまかせる、という書き方である。

クラスにコンストラクタを一つでも明示的に宣言すると、引数なしのコンストラクタは暗黙には宣言されない。基本編の constructor-destructor の章で、`Counter(int)` を書いたあとに引数なしの `Counter c;` がビルドに失敗したのは、これによる。そのとき足した引数なしのコンストラクタは、`= default` でも書ける。

```cpp
class Point
{
    int x;
    int y;

public:
    Point(int a, int b) : x(a), y(b) {}
    Point() = default;
};
```

`Point` でも `Point(int, int)` を明示的に宣言したので、引数なしの `Point` は暗黙には宣言されない。`Point() = default;` がなければ `Point p;` は呼べる関数がなく、ビルドに失敗する。`Point() = default;` は、引数なしのコンストラクタを明示的に宣言し、その定義を自動で生成させる。これで `Point p;` も `Point q(3, 4);` も書ける。

## = delete

`= delete` は、関数を禁じる。禁じた関数を呼ぶと、ビルドに失敗する。

例として、`int` を受け取る `process` と、`double` を受け取る `process` があるとする。`double` 版は禁じておく。

```cpp
void process(int x) { std::println("int: {}", x); }
void process(double) = delete;
```

`process(5)` は `int` 版が呼ばれる。`process(3.14)` は禁じた版が選ばれる。

:::message
禁じた関数を呼ぶ
```cpp
process(3.14);
```
`3.14` は `double` なので、オーバーロード解決では `process(double)` が選ばれる。だが `process(double)` は `= delete` されているので、これを呼ぶことはできず、ビルドに失敗する。
:::

`= delete` した関数も、オーバーロード解決では候補として選ばれる。コンセプトの章で見た、制約を満たさず候補から外れるのとは異なり、`= delete` した関数は候補に残り、選ばれたうえで「呼べない」としてビルドに失敗する。

## 暗黙に delete される特殊メンバ関数

`= delete` と書かなくても、特殊メンバ関数が自動で delete されることがある。自動で定義しようとして、その定義が成り立たない場合である。

たとえば、参照をメンバに持つ型を考える。

```cpp
struct Holder
{
    int& ref;
};
```

参照は宣言時に初期化が要る(基本編の参照とポインタの章)。引数なしのコンストラクタは `ref` を初期化しようがなく、その定義は成り立たない。このとき、引数なしのコンストラクタは暗黙に宣言されるが、delete されたものとして定義される。

:::message
delete された引数なしコンストラクタを呼ぶ
```cpp
Holder h;
```
`Holder` の引数なしコンストラクタは delete されたものとして定義されている。`Holder h;` はそれを呼ぶので、`= delete` した関数を呼ぶのと同じく、ビルドに失敗する。
:::

`= default` の節で見た `Point` は、引数なしのコンストラクタが暗黙には宣言されず、「呼べる関数がない」のだった。`Holder` では、引数なしのコンストラクタは暗黙に宣言されたうえで delete されている。定義が成り立たないとき、特殊メンバ関数は `= delete` と書かずとも、このように自動で delete される。

これは `= default` と書いても変わらない。`Holder` に `Holder() = default;` と明示しても、その定義は成り立たないので、やはり delete されたものとして定義される。`= default` は定義を自動生成にまかせる指示であって、成り立たない定義を成り立たせるものではない。

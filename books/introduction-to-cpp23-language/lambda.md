---
title: "ラムダ"
---

これまで関数は、名前を付けて定義してきた。名前を付けず、その場で書く関数が **ラムダ** である。本章では、ラムダの書き方、外側の変数を取り込むキャプチャ、参照キャプチャと寿命の関わり、そして引数の型を `auto` にするジェネリックラムダを扱う。

## ラムダ

値を二乗するラムダを書く。

```cpp
auto square = [](int x) { return x * x; };
std::println("{}", square(5));   // 25
```

ラムダは `[](引数) { 本体 }` の形で書く。`[]` はキャプチャ(次の節で扱う)、`()` は引数、`{}` は本体である。戻り値の型は、戻り値の型を `auto` としたときと同じく、`return` する式から決まる。`square(5)` は `5 * 5` で `25` になる。

ラムダ式からは、関数のように呼び出せるオブジェクトが生まれる。これを **クロージャ** と呼ぶ。クロージャは、`operator()` を持つ。`operator()` は関数呼び出しの `()` を定義するメンバ関数で、演算子オーバーロードの章で、メンバとしてしか定義できない演算子の一つに挙げた。`square` は、おおよそ次のような、その `operator()` を持つオブジェクトと同じである。

```cpp
struct
{
    int operator()(int x) const { return x * x; }
} square;
```

ラムダ式ごとに、固有の名前を持つ型のオブジェクトが作られる。その名前は自分で書くことはできないので、`auto` で受けて、何度でも呼ぶ。

:::details キャプチャのないラムダは static にできる(C++23)
演算子オーバーロードの章で見たとおり、関数呼び出しの `()` は C++23 から静的メンバにもできる。キャプチャのないラムダでも、`operator()` を `static` にできる。`static` を付けると、呼び出しのときにクロージャのオブジェクトを渡さない。最初の `square` なら、こう書ける。

```cpp
auto square = [](int x) static { return x * x; };
```

このとき `operator()` は、静的メンバの章で見た静的メンバ関数で、おおよそ `static int operator()(int x) { return x * x; }` にあたる。キャプチャがなければ、保持するものがなく、呼び出しに使うオブジェクトもいらないので、それを渡さずに済む。`static` を書けるのはキャプチャのないラムダだけで、キャプチャするものがあるラムダには付けられない。
:::

## キャプチャ

ラムダの本体から外側の変数を使うには、その変数を **キャプチャ** する。`[]` の中に書く。

倍率 `factor` をキャプチャして、引数を `factor` 倍するラムダを書く。

```cpp
int factor = 3;
auto mul = [factor](int x) { return x * factor; };
std::println("{}", mul(5));   // 15
```

`[factor]` は、その時点の `factor` の値をクロージャにコピーして保持する。これを **値キャプチャ** と呼ぶ。`mul` は、おおよそ次のように、`factor` のコピーをデータメンバに持つ。

```cpp
struct
{
    int factor;
    int operator()(int x) const { return x * factor; }
} mul = {.factor = factor};
```

`[&factor]` のように `&` を付けると、コピーではなく、外側の変数そのものを参照で保持する。これを **参照キャプチャ** と呼ぶ。参照キャプチャしたラムダは、外側の変数を読み書きできる。

```cpp
int sum = 0;
auto add = [&sum](int x) { sum += x; };
add(5);
add(3);
std::println("{}", sum);   // 8
```

`add` は `sum` を参照キャプチャしているので、`add(5)` と `add(3)` はどちらも外側の `sum` を書き換える。最後に `sum` は `8` になる。`add` は、おおよそ次のように、`sum` への参照をデータメンバに持つ。

```cpp
struct
{
    int& sum;
    void operator()(int x) const { sum += x; }
} add = {.sum = sum};
```

`operator()` は `const` だが、メンバ `sum` は参照なので、その参照先である外側の `sum` を書き換えられる。値キャプチャがコピーを持つのに対し、参照キャプチャは参照を持つ、という違いがここに現れる。

## 参照キャプチャと寿命

参照キャプチャは、外側の変数そのものを指す。その変数が先に寿命を終えると、ラムダは消えたオブジェクトを指すことになる。

:::message alert
寿命の切れた変数を参照キャプチャする
```cpp
auto make()
{
    int local = 10;
    return [&local]() { return local; };
}
```
`local` は `make` を抜けるときに破棄される。返ってきたラムダを呼ぶと、すでに消えた `local` を参照することになり、結果は定まらず、実行時に深刻な問題が起こりうる。
:::

`local` のような自動記憶域期間のオブジェクトがブロックを抜けると破棄されることは、寿命と記憶域期間の章で見た。参照キャプチャがその破棄されたオブジェクトを指すと、この問題が起きる。参照キャプチャするときは、キャプチャした変数がラムダより長く生きることを確かめる。

## ジェネリックラムダ

引数の型を `auto` にすると、いろいろな型に使えるラムダになる。これを **ジェネリックラムダ** と呼ぶ。

```cpp
auto dbl = [](auto x) { return x * 2; };
std::println("{}", dbl(5));     // 10
std::println("{}", dbl(1.2));   // 2.4
```

`dbl` は、おおよそ次のように、`operator()` を関数テンプレートとして持つ。

```cpp
struct
{
    template <typename T>
    auto operator()(T x) const { return x * 2; }
} dbl;
```

`auto` の引数は、テンプレートの型パラメータになる。だから、テンプレートの章で見た実引数推論のとおり、呼び出しごとに実引数から型が決まる。`dbl(5)` では `T` が `int`、`dbl(1.2)` では `T` が `double` になる。

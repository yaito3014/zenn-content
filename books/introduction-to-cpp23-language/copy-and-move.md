---
title: "コピーとムーブ"
---

基本編で、オブジェクトをコピーする場面(値渡しや、仮想関数の章のスライシング)を見た。コピーは複製を作る。だが、もとが二度と使われない一時的な値なら、複製する代わりに中身を移してしまえる。これを **ムーブ** と呼ぶ。本章では、コピーとムーブを担う特殊メンバ関数、ムーブを促す `std::move`、コピーを許さないムーブのみの型を扱う。

## 特殊メンバ関数とコピー

`int` の配列を持つ `Buffer` を考える。コンストラクタで `new[]` で確保し、デストラクタで `delete[]` する。

```cpp
class Buffer
{
    int* data;
    int size;

public:
    explicit Buffer(int n) : data(new int[n]), size(n) {}
    ~Buffer() { delete[] data; }
};
```

オブジェクトをコピーすると、特殊メンバ関数の一つ、**コピーコンストラクタ** が呼ばれる。自分で書かなければ、メンバをそのまま写すコピーコンストラクタが使われる。だが、ポインタを持つ `Buffer` では、それでは正しく複製できない。

:::message alert
ポインタを持つ型を、メンバをそのまま写してコピーする
```cpp
Buffer a(10);
Buffer b = a;
```
メンバがそのまま写ると、ポインタ `data` の値(アドレス)も写り、`a` と `b` は同じ配列を指す。両方のデストラクタが同じ配列を `delete[]` し、二重に解放される。実行時に深刻な問題が起こりうる。
:::

配列ごと複製するには、コピーコンストラクタを自分で書く。

```cpp
Buffer(const Buffer& other) : data(new int[other.size]), size(other.size)
{
    for (int i = 0; i < size; i = i + 1)
        data[i] = other.data[i];
}
```

新しく配列を確保し、中身を写す。これで `a` と `b` は別々の配列を持ち、それぞれが自分の配列を `delete[]` する。

## ムーブ

コピーは配列ごと複製する。だが、もとの `Buffer` が二度と使われないなら、複製せずに、その配列を新しい `Buffer` へ移してしまえる。これを担うのが **ムーブコンストラクタ** である。

```cpp
Buffer(Buffer&& other) : data(other.data), size(other.size)
{
    other.data = nullptr;
    other.size = 0;
}
```

`other` の配列を、新しい `Buffer` がそのまま受け取る(`data` にアドレスを写すだけで、確保はしない)。そのうえで `other.data` を `nullptr` にする。こうすると、`other` のデストラクタは `delete[] nullptr` となる。`delete[]` に `nullptr` を渡しても何も起こらないので、移したあとの配列を二重に解放しない。

ムーブコンストラクタの引数は、rvalue 参照 `Buffer&&` である。値カテゴリの章で見たとおり、rvalue 参照は rvalue に束縛する。rvalue は `const Buffer&` にも束縛できるが、`Buffer&&` があればそちらが優先され、ムーブコンストラクタが選ばれる。

### std::move

名前のついた `Buffer`(lvalue)も、もう使わないと分かっているなら、ムーブの対象にしたい。lvalue を rvalue として扱わせるのが **`std::move`** である。

```cpp
Buffer a(10);
Buffer b = a;             // コピー。a は lvalue
Buffer c = std::move(a);  // ムーブ。std::move(a) は rvalue
```

`Buffer b = a;` では `a` が lvalue なので、コピーコンストラクタが呼ばれる。`std::move(a)` は `a` を rvalue として扱わせるので、`Buffer c = std::move(a);` ではムーブコンストラクタが呼ばれる。`std::move` 自体は、何も移さない。値カテゴリを変えるだけで、実際に配列を移すのはムーブコンストラクタである。

ムーブしたあとの `a` は、配列を手放して空になっている(`data` は `nullptr`)。`a` を壊すこと(デストラクタ)は安全だが、移したあとの中身をあてにして使うことはできない。

:::details ムーブできないメンバと、除外される自動のムーブ
ムーブコンストラクタを書かなければ、自動で定義される。だが、メンバのどれかがムーブできないと、その自動のムーブコンストラクタは、`= default` と `= delete` の章で見た「暗黙に delete される」と同じく、delete されたものとして定義される。

```cpp
struct OnlyCopy
{
    OnlyCopy() = default;
    OnlyCopy(const OnlyCopy&) {}
    OnlyCopy(OnlyCopy&&) = delete;   // ムーブを禁じる
};

struct Wrap
{
    OnlyCopy m;
};
```

`Wrap` のメンバ `m` はムーブできないので、`Wrap` の自動のムーブコンストラクタは delete されたものとして定義される。`= default` と `= delete` の章では、暗黙に delete された特殊メンバ関数は候補に残り、選ばれたうえでビルドに失敗するのだった。だが、ムーブコンストラクタはこれと異なり、delete されたものとして定義されると、オーバーロード解決の候補から除外される。`= delete` した関数が候補に残るのとも異なる。

```cpp
Wrap a;
Wrap b = std::move(a);
```

そのため、`Wrap b = std::move(a);` はムーブコンストラクタが候補にならず、代わりにコピーコンストラクタが使われる。ムーブできないメンバを持つ型でも、コピーはできる。
:::

## ムーブのみの型

コピーすれば複製ができる。だが、ある資源をただ一つのオブジェクトだけに持たせたい型もある。複製されては、その「ただ一つ」が崩れる。コピーを禁じるには、コピーコンストラクタを `= delete` する。

```cpp
Buffer(const Buffer& other) = delete;
```

`Buffer` には、先に書いたムーブコンストラクタがある。コピーコンストラクタを `= delete` しても、そのムーブコンストラクタは残るので、`Buffer` はムーブはできるが、コピーはできない。

:::message
コピーを禁じた型をコピーする
```cpp
Buffer a(10);
Buffer b = a;
```
コピーコンストラクタが `= delete` されているので、`Buffer b = a;` はビルドに失敗する。`Buffer c = std::move(a);` のようにムーブはできる。
:::

:::details コピーを delete しただけではムーブは残らない
`Buffer` がムーブできるのは、ムーブコンストラクタを自分で書いたからである。ムーブコンストラクタを書いていない型で、コピーコンストラクタを宣言する(`= delete` も宣言である)と、自動のムーブコンストラクタは宣言されない。そのような型では、`std::move` してもムーブコンストラクタがなく、`= delete` したコピーコンストラクタが選ばれて、ビルドに失敗する。コピーを禁じてムーブだけを許すには、ムーブコンストラクタを自分で書いておく必要がある。
:::

資源をただ一つのオブジェクトに持たせる型は、標準ライブラリにもある。それはこの先の編で扱う。

:::details 特殊メンバ関数の一覧
特殊メンバ関数には、本章で書いたコピーコンストラクタ・ムーブコンストラクタ・デストラクタのほかに、代入のための **コピー代入演算子**・**ムーブ代入演算子**、引数なしで作る **デフォルトコンストラクタ** がある。資源を持つ型では、コピーとムーブの構築・代入とデストラクタを揃えて定義することが多い。
:::

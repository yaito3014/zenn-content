---
title: "抽象クラスとインターフェース"
---

基本編の仮想関数の章で、基底クラスの仮想関数を派生クラスでオーバーライドし、基底クラスの参照やポインタを通して、派生クラスごとの振る舞いを呼び分けた。そこでは、基底クラスの仮想関数にも実装があった。本章では、実装を派生クラスにまかせる **純粋仮想関数**、それを持つ **抽象クラス**、そして操作の約束だけを表す **インターフェース** を扱う。

## 純粋仮想関数

基底クラスの仮想関数に、意味のある実装を書けないことがある。たとえば、図形の面積を返す関数を考える。「図形」一般には決まった面積の求め方がなく、正方形なら一辺の二乗、長方形なら幅と高さの積、と派生クラスごとに違う。基底クラスの側では、面積を返すという約束だけを示し、求め方は派生クラスにまかせたい。

このような、実装を派生クラスにまかせる仮想関数を **純粋仮想関数** と呼ぶ。仮想関数の宣言の後ろに `= 0` を付けて書く。

```cpp
class Shape
{
public:
    virtual int area() const = 0;
    virtual ~Shape() = default;
};
```

`virtual int area() const = 0;` が純粋仮想関数である。`= 0` は、実装をここで与えず、派生クラスでのオーバーライドにまかせることを表す。`= default` と `= delete` の章で見た `=` の付いた宣言と形は似ているが、`= 0` は仮想関数を純粋仮想関数にするための書き方である。

デストラクタを `virtual` にしているのは、基本編の仮想関数の章で見たとおり、基底クラスのポインタを通してオブジェクトを破棄するときに、派生クラスのデストラクタを正しく呼ぶためである。

純粋仮想関数を持つクラスを **抽象クラス** と呼ぶ。抽象クラスは、それ自体のオブジェクトを作れない。

:::message
抽象クラスのオブジェクトを作る
```cpp
Shape s;
```
`Shape` は純粋仮想関数 `area` を持つ抽象クラスである。抽象クラスそのもののオブジェクトは作れず、ビルドに失敗する。
:::

## 派生クラスで実装する

抽象クラスは、派生クラスのための土台になる。派生クラスで純粋仮想関数をオーバーライドすると、その派生クラスはオブジェクトを作れる。

```cpp
class Square : public Shape
{
public:
    explicit Square(int s) : side(s) {}
    int area() const override { return side * side; }

private:
    int side;
};

class Rectangle : public Shape
{
public:
    Rectangle(int w, int h) : width(w), height(h) {}
    int area() const override { return width * height; }

private:
    int width;
    int height;
};
```

`Square` と `Rectangle` は、`Shape` の純粋仮想関数 `area` を、それぞれの求め方でオーバーライドした。これで両方ともオブジェクトを作れる。

基底クラスの参照を通して `area` を呼ぶと、基本編で見たとおり、動的な型に応じたオーバーライド先が呼ばれる。

```cpp
void print_area(Shape const& s)
{
    std::println("{}", s.area());
}
```

```cpp
Square sq(4);
Rectangle re(3, 5);
print_area(sq);   // 16
print_area(re);   // 15
```

`print_area` は `Shape const&` を受け取る。`Shape` のオブジェクトは作れないが、`Shape` への参照は作れる。`print_area(sq)` では動的な型が `Square` なので `Square::area` が呼ばれ、`16` を出力する。`print_area(re)` では `Rectangle::area` が呼ばれ、`15` を出力する。

## インターフェース

`Shape` は、データを持たず、操作 `area` を純粋仮想関数として約束するだけのクラスである。`Shape` を継承する型は、`area` を備えることを約束する。

このように、操作の約束だけを表す抽象クラスを **インターフェース** と呼ぶ。C++ には、インターフェースを表す専用のキーワードはない。純粋仮想関数で操作を約束する抽象クラスが、その役割を果たす。

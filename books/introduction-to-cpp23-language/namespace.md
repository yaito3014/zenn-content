---
title: "名前空間"
---

基本編では、`std::println` のように名前の頭に `std::` を付けて使ってきた。この `std::` が何かは、名前の一部として受け取るにとどめていた。`std` は **名前空間** であり、標準ライブラリは自身の名前のほとんどをこの名前空間の中で宣言している。`std::println` は、名前空間 `std` の中の `println` を指す。

名前空間は、宣言をまとめ、名前を区切るための仕組みである。同じ名前が別々の用途でぶつからないように分ける。`std` のように既にあるものを使うだけでなく、名前空間は自分でも定義できる。

## 名前空間を定義する

`namespace` に続けて名前と `{ }` を書き、その中に宣言を並べる。

```cpp
namespace geo
{
    struct Point
    {
        int x;
        int y;
    };

    int sum(Point p)
    {
        return p.x + p.y;
    }
}
```

これは `geo` という名前空間を定義し、その中に `Point` と `sum` を宣言する。名前空間の中の名前を外から使うには、`geo::Point` のように名前空間の名前と `::` を前に付ける。

```cpp
geo::Point p = {.x = 3, .y = 4};
std::println("{}", geo::sum(p));
```

これは `7` と出力する。`geo::` を前に付けた `geo::Point` や `geo::sum` のような名前を **修飾名** と呼ぶ。基本編から使ってきた `std::println` も、名前空間 `std` の `println` を修飾名で書いたものである。

## 名前の探索

名前を書いたとき、それがどの宣言を指すかは、決められた範囲を探して決まる。`geo::Point` のように名前空間を修飾した修飾名は、その名前空間の中を探す。一方、`Point` のように修飾しない名前(**非修飾名**)は、いま書いている場所から外側の名前空間へ向かって探すが、`geo` のような別の名前空間の中までは入っていかない。

そのため、`geo` の外で `Point` を修飾せずに書くと、その名前は見つからない。

:::message
名前空間の中の型を修飾せずに使う
```cpp
Point p = {.x = 3, .y = 4};
```
`Point` は名前空間 `geo` の中で宣言されている。`geo` の外で `Point` と修飾せずに書くと、この名前は見つからず、ビルドに失敗する。
:::

毎回 `geo::` を付ける代わりに、使いたい名前を **using 宣言** で今の場所に持ち込める。

```cpp
using geo::Point;

Point p = {.x = 3, .y = 4};
```

`using geo::Point;` と書くと、以降は `Point` だけで `geo::Point` を指せる。

:::details using 宣言と using ディレクティブ
`using geo::Point;` は名前を1つだけ持ち込む。これに対し `using namespace geo;` と書くと、`geo` の中のすべての名前を一度に持ち込む(**using ディレクティブ**)。このとき、持ち込んだ名前が別の名前とぶつかると、どちらを指すか決められず、ビルドに失敗することがある。本書では、必要な名前を `using geo::Point;` で1つずつ持ち込む形を使う。
:::

## 引数依存の名前探索

ところが、関数の呼び出しには、もう一つの探索が加わる。`sum` は `geo` の中で宣言されているのに、`geo` の外で修飾せずに呼び出せる。

```cpp
geo::Point p = {.x = 3, .y = 4};
std::println("{}", sum(p));
```

`sum(p)` と修飾せずに書いても、これは `geo::sum` を指し、`7` と出力する。実引数 `p` の型が `geo::Point` であるため、`geo` の中も `sum` の探索の対象になる。このように、関数を非修飾で呼び出すとき、実引数の型が属する名前空間も探索されることを **引数依存の名前探索**(argument-dependent lookup、ADL)と呼ぶ。

型の名前には、この探索は働かない。そのため、`Point` は修飾しないと見つからないのに、`sum(p)` は修飾しなくても見つかる。

ADL により、引数の型に結びついた関数を、名前空間を修飾せずに呼べる。標準ライブラリにも、この仕組みで修飾なしに使える名前があるが、その具体的な例はライブラリ編で扱う。基本編で名前の一部として受け取っていた `std::` の `std` も、ここで見た名前空間の1つにほかならない。

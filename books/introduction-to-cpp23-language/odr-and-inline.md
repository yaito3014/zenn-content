---
title: "ODR と inline"
---

基本編の複数ファイルの章で、関数の宣言は何度でも書けるが、定義は1つだけ書く、と見た。定義が見つからないとリンクに失敗した。この「定義は1つ」という決まりを **One Definition Rule**(ODR)と呼ぶ。本章では、ODR をもう少し詳しく見て、それを緩める `inline` を扱う。

## 定義はひとつ

同じ関数について、宣言は何度でも書ける。だが、その定義はプログラム全体で1つでなければならない。複数ファイルの章では、定義が1つもないとき、リンクで定義が見つからなかった。逆に、定義が2つあると、リンクで定義が重複する。

## ヘッダに定義を置くと

定義の重複は、ヘッダに定義を書いたときに起こりやすい。次のヘッダは、`square` の定義を含む。

```cpp
// square.hpp
int square(int x)
{
    return x * x;
}
```

このヘッダを複数のソースファイルが `#include` すると、取り込んだそれぞれの翻訳単位に `square` の定義が現れる。プログラム全体では `square` の定義が2つ以上になり、ODR に反する。

:::message
ヘッダに置いた定義を複数のソースから取り込む
```cpp
// square.hpp
int square(int x) { return x * x; }

// a.cc
#include "square.hpp"

// b.cc
#include "square.hpp"
```
`square` の定義が `a.cc` と `b.cc` の両方の翻訳単位に現れ、プログラム全体で定義が重複する。リンクに失敗する。
:::

## inline で定義を複数の翻訳単位に置く

定義に `inline` を付けると、同一の定義が複数の翻訳単位に現れてよくなる。

```cpp
// square.hpp
inline int square(int x)
{
    return x * x;
}
```

`inline` を付けた `square` は、複数のソースから取り込んでも、定義の重複とはみなされない。ただし、どの翻訳単位に現れる定義も同一でなければならない。これで、関数の定義をヘッダに置ける。

:::message alert
定義が翻訳単位ごとに食い違う
`inline` を付けても、定義はどの翻訳単位でも同一でなければならない。同じ名前の `inline` 関数を、ソースごとに違う中身で定義してしまうと、この決まりに反する。これはビルドが通ってしまうことがあり、どの定義が使われるかも定まらず、実行時に深刻な問題が起こりうる。
:::

`inline` という名は、かつて呼び出しをその場に展開することを示唆したが、現在の主な役割は、こうして定義の重複を許すことである。

基本編では、メンバ関数 `area` の本体をクラスの中に書き、そのまま使えた。クラスの中で定義したメンバ関数は、暗黙に `inline` として扱われる。複数のソースから取り込んでも定義の重複とならないのは、`inline` を付けた関数と同じ理由による。

`inline` は、関数だけでなく変数にも付けられる。ヘッダに変数の定義を置くと、関数と同じく複数の翻訳単位に定義が現れて ODR に反する。`inline` を付けると、それらは重複とはみなされず、プログラム全体で1つの変数を指すものとして扱われる。

```cpp
// config.hpp
inline int version = 23;
```

`inline` を付けた `version` は、複数のソースから取り込んでも、どこでも同じ一つの変数を指す。これで、変数の定義もヘッダに置ける。

:::details リンケージ
名前には、翻訳単位をまたいで同じ実体を指すものと、その翻訳単位の中だけに閉じるものがある。前者を **外部リンケージ**、後者を **内部リンケージ** という。関数や名前空間スコープの変数は外部リンケージを持つことが多く、ODR と `inline` はそうした名前についての話である。一方、名前空間スコープに `static` を付けた名前や、無名名前空間(`namespace { ... }`)の中の名前は内部リンケージを持ち、翻訳単位ごとに別の実体になる。そのため、同じ名前を別々のソースに書いても重複しない。
:::

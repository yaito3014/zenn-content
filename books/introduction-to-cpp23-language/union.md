---
title: "共用体"
---

メモリレイアウトの章で、構造体のメンバはそれぞれの記憶域を持ち、宣言した順に並ぶと見た。これに対し、複数のメンバが一つの記憶域を分け合い、そのうち一つだけを保持するのが **共用体**(union)である。本章では、共用体と、有効なメンバを管理するタグ付き共用体を扱う。

## 共用体

`int` または `double` のどちらかを保持する共用体を定義する。

```cpp
union Value
{
    int i;
    double d;
};
```

構造体と違い、共用体のメンバはすべて同じ記憶域に重なって置かれる。だから、一度に保持できるのは一つのメンバだけである。

```cpp
Value v;
v.i = 42;
std::println("{}", v.i);   // 42
```

メンバが記憶域を分け合うので、共用体の大きさは、最も大きいメンバを収められる大きさになる。ここで、`int` が4バイト、`double` が8バイトの環境を考える。このとき `Value` で最も大きいメンバは `double` である。

```cpp
std::println("{}", sizeof(Value));   // 8
```

`sizeof(Value)` は、`int` と `double` を足した12ではなく、最も大きい `double` の8になる。

## 有効でないメンバを読む

共用体で、最後に値を入れたメンバを **有効なメンバ** と呼ぶ。有効でないメンバを読んではならない。

:::message alert
有効でないメンバを読む
```cpp
Value v;
v.i = 42;
std::println("{}", v.d);
```
`v.i = 42` で有効なメンバは `i` になっている。有効でない `d` を読むと、結果は定まらず、実行時に深刻な問題が起こりうる。
:::

共用体は、どのメンバが有効かという情報を持たない。有効なメンバは、使う側が管理する。

## タグ付き共用体

どのメンバが有効かを覚えておくために、共用体と、それを示す印を組み合わせる。印には、基本編の列挙型を使う。

```cpp
enum class Tag { integer, real };

struct Number
{
    Tag tag;
    union
    {
        int i;
        double d;
    };
};
```

メンバに名前を付けない共用体(無名共用体)を構造体の中に置くと、その共用体のメンバを `n.i` のように構造体のメンバとして書ける。`tag` に、いま有効なメンバを記録する。

```cpp
void print(Number const& n)
{
    if (n.tag == Tag::integer)
    {
        std::println("{}", n.i);
    }
    else
    {
        std::println("{}", n.d);
    }
}
```

```cpp
Number a = {.tag = Tag::integer, .i = 42};
Number b = {.tag = Tag::real, .d = 3.5};
print(a);   // 42
print(b);   // 3.5
```

`tag` を見てから、それに合うメンバだけを読む。こうして有効なメンバを `tag` で管理する形を **タグ付き共用体** と呼ぶ。

標準ライブラリには、有効なメンバの管理を引き受ける、より安全な道具 `std::variant` がある。これはこの先の編で扱う。

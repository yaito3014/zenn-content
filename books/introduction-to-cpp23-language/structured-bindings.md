---
title: "構造化束縛"
---

複数の値をまとめた構造体は、`p.x` のようにメンバごとにアクセスして使う。まとめられた各部分に、一度の宣言で名前を付けられるのが **構造化束縛** である。本章では、構造体と配列の構造化束縛を扱う。

## 構造化束縛

たとえば、割り算の商と余りをまとめて返す関数を考える。商と余りを `DivResult` にまとめて返す。

```cpp
struct DivResult
{
    int quotient;
    int remainder;
};

DivResult divide(int a, int b)
{
    return {.quotient = a / b, .remainder = a % b};
}
```

戻り値を受けるとき、構造化束縛で商と余りに別々の名前を付けられる。

```cpp
auto [q, r] = divide(17, 5);
std::println("{} {}", q, r);   // 3 2
```

`auto [q, r]` は、右辺の結果を一つのオブジェクトとしてコピーし、その各メンバに前から順に `q`・`r` という名前を付ける。`q` が `quotient`、`r` が `remainder` にあたる。構造体に構造化束縛を使うには、それを書く場所から、構造体のメンバすべてにアクセスできなければならない。名前の数は、メンバの数と同じでなければならない。

:::message
メンバの数と名前の数が合わない
```cpp
auto [a, b, c] = divide(17, 5);
```
`DivResult` のメンバは `quotient` と `remainder` の二つで、名前を三つ書くと合わない。ビルドに失敗する。
:::

## 参照で束縛する

`auto [q, r]` は右辺をコピーするので、付けた名前を書き換えても、もとのオブジェクトには届かない。`auto` に `&` を付けて `auto&` とすると、コピーせず、もとのオブジェクトを参照で束縛する。各名前は、もとのオブジェクトのメンバそのものを指す。

```cpp
struct Point
{
    int x;
    int y;
};

Point p = {.x = 1, .y = 2};
auto& [a, b] = p;
a = 10;
std::println("{} {}", p.x, p.y);   // 10 2
```

`auto& [a, b]` で、`a` は `p.x` を、`b` は `p.y` を指す。`a = 10` は `p.x` を書き換える。書き換えずに読むだけなら、`auto const&` で束縛できる。`const` なので書き換えはできず、コピーも避けられる。

## 配列の構造化束縛

構造化束縛は、配列にも使える。各要素に前から順に名前が付く。

```cpp
int arr[3] = {4, 5, 6};
auto [x, y, z] = arr;
std::println("{} {} {}", x, y, z);   // 4 5 6
```

`x`・`y`・`z` が、`arr` の要素 `4`・`5`・`6` にあたる。配列でも、名前の数は要素の数と同じでなければならない。

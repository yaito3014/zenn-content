---
title: "テンプレート"
---

基本編では、`std::vector<int>` の `<int>` を名前の一部として使うにとどめた。この `<int>` は、`vector` という型に `int` を与えたものである。パラメータをとり、与えられたパラメータごとに具体的な関数やクラスになる宣言を **テンプレート** と呼ぶ。テンプレートにできる宣言はいくつかあり、本章では、そのうち型をパラメータにとる関数・クラス・型エイリアスのテンプレートを扱う。

## 関数テンプレート

基本編の関数は、`int` のような特定の型に対して書いた。同じ処理を別の型にも使いたいと、型ごとに書き直すことになる。テンプレートは、型をパラメータにして、その処理を1つだけ書く。

```cpp
template <typename T>
T twice(T x)
{
    return x + x;
}
```

`template <typename T>` は、`T` という型のパラメータを導入する。本体では `T` を型として使える。これを **関数テンプレート** と呼ぶ。

`class T` と書いても同じだが、本書は `typename` を使う。

呼び出すと、実引数の型から `T` が決まる。

```cpp
twice(5);     // T は int。10 を返す
twice(2.5);   // T は double。5.0 を返す
```

`twice(5)` では実引数 `5` が `int` なので `T` は `int` に、`twice(2.5)` では `double` になる。実引数から型パラメータが決まることを **実引数推論** と呼ぶ。`twice<int>(5)` と型を明示することもできるが、推論できるなら書かなくてよい。

## クラステンプレート

クラスも、型をパラメータにできる。

```cpp
template <typename T>
struct Box
{
    T value;
};
```

これを **クラステンプレート** と呼ぶ。使うときは、`Box<int>` のように型を与える。

```cpp
Box<int> a = {3};
Box<double> b = {2.5};
```

`Box<int>` は `value` が `int`、`Box<double>` は `value` が `double` になる。基本編で名前の一部として使った `std::vector<int>` の `<int>` も、これと同じく、クラステンプレート `vector` に型 `int` を与えたものである。標準ライブラリが提供する型の多くは、こうしたテンプレートである。

:::details クラステンプレートの型引数の推論
クラステンプレートでも、初期化子から型引数を推論できる場合がある(クラステンプレートの実引数推論、C++17 以降)。本章では `Box<int>` と型を明示する形を使う。
:::

## エイリアステンプレート

型エイリアスも、型をパラメータにできる。

```cpp
template <typename T>
using Ptr = T*;
```

`Ptr<int>` は `int*`、`Ptr<double>` は `double*` の別名になる。型エイリアスの章で見た `using` に、テンプレートのパラメータを加えた形である。

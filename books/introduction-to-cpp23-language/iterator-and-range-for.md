---
title: "イテレータと範囲for"
---

基本編の配列の章で、`for (int i = 0; i < 5; i = i + 1)` のように添字を動かして、配列の各要素を順に扱った。要素を先頭から順に見るだけなら、添字を介さずに書く **範囲for** がある。本章では、範囲for と、それを支える **イテレータ** と **センチネル** を扱う。

## 範囲for

配列の各要素を順に出力する。

```cpp
int arr[] = {1, 2, 3, 4, 5};
for (int x : arr)
{
    std::println("{}", x);
}
```

`for (int x : arr)` は、`arr` の要素を先頭から順に一つずつ `x` に取り出し、続く `{ }` の中を実行する。添字 `i` も大きさ `5` も書かない。これは `1` から `5` までを順に出力する。

## 自分の型を範囲for で回す

範囲for は配列だけのものではない。自分で定義した型でも、`begin` と `end` という二つのメンバ関数を用意すれば、範囲for で回せる。

例として、`first` から `last` の手前までの整数を順に与える `IntRange` を作る。

```cpp
struct IntRange
{
    int first;
    int last;

    struct sentinel
    {
        int bound;
    };

    struct iterator
    {
        int value;
        int operator*() const { return value; }
        void operator++() { ++value; }
        bool operator==(sentinel const& s) const { return value == s.bound; }
    };

    iterator begin() const { return {.value = first}; }
    sentinel end() const { return {.bound = last}; }
};
```

`begin` は先頭の `first` から始まるものを、`end` は終端の `last` を表すものを返す。その二つの型 `iterator` と `sentinel` は、`IntRange` の中に定義した。このように、クラスの中には別のクラスを定義できる。これで `IntRange` を範囲for で回せる。

```cpp
for (int x : IntRange{.first = 1, .last = 5})
{
    std::println("{}", x);
}
```

これは `1` から `4` までを順に出力する。`last` の `5` は手前までなので含まれない。

## イテレータとセンチネル

範囲for は、先頭から終端まで、要素を一つずつたどる。先頭から終端へ進んでいく側を **イテレータ**、終端を表す印を **センチネル** と呼ぶ。`begin` がイテレータを、`end` がセンチネルを返す。

イテレータは、要素をたどる操作を備える。

- `*` で、いまの位置の要素を取り出す
- `++` で、次の位置へ進める

センチネルは終端を表すもので、要素を持つとは限らない。イテレータが終端に達したかは、`!=` で比べる。達していない間は `!=` が成り立ち、繰り返しが続く。

演算子オーバーロードの章で見たように、演算子は `operator` に記号を続けた名前の関数として定義する。`iterator` には、要素を取り出す `operator*`、次へ進める `operator++`、そしてセンチネルと比べる `operator==` を定義した。`operator==` は、イテレータと `sentinel` を比べ、`value` が終端に達したかを返す。`!=` は定義していないが、演算子オーバーロードの章で見たとおり、`operator==` を定義すると `!=` はその否定として使えるようになる。

イテレータとセンチネルは、別の型でよい。イテレータは要素をたどる操作を備え、センチネルは終端を表す。両者を同じ型にする必要はない。範囲for が `begin` と `end` に別の型を許すのは C++17 から、終端のこの役割をセンチネルと呼ぶ概念が整ったのは C++20 からである。

範囲for は、これらの操作を使った繰り返しと等価である。先ほどの

```cpp
for (int x : IntRange{.first = 1, .last = 5})
{
    std::println("{}", x);
}
```

は、次の形に展開される。

```cpp
{
    auto&& __range = IntRange{.first = 1, .last = 5};
    auto __begin = __range.begin();   // イテレータ
    auto __end = __range.end();       // センチネル
    for (; __begin != __end; ++__begin)
    {
        int x = *__begin;
        std::println("{}", x);
    }
}
```

`auto&& __range` の `auto&&` は **転送参照** と呼ばれる参照で、名前付きの範囲でも一時オブジェクトでも、コピーせずに受けられる(下のコラムで扱う)。`IntRange{.first = 1, .last = 5}` のような一時オブジェクトを渡しても、値カテゴリの章で `int const&` について見たように、その一時オブジェクトは `__range` が使えるあいだ保たれる。`__range`・`__begin`・`__end` は、展開のしくみを示すための名前で、自分で書くものではない。`__begin` を先頭から始め、センチネル `__end` に達するまで `++` で一つずつ進める。各回で `*__begin` から要素を取り出して `x` に入れ、本体を実行する。`IntRange{.first = 1, .last = 5}` では、`__begin` の `value` が `1` から始まり、`5` になった時点で `__end` と等しくなって終わる。だから出力は `4` までになる。

:::details auto&& と転送参照
脱糖形の `auto&& __range` の `auto&&` は、ふつうの参照とは違う。束縛するものが lvalue なら lvalue 参照に、rvalue なら rvalue 参照になる。このように、初期化子の値カテゴリによって参照の型が決まる参照を **転送参照**(forwarding reference)と呼ぶ。ユニバーサル参照とも呼ばれる。

値カテゴリの章で見た `int const&` も lvalue・rvalue のどちらにも束縛できたが、あれは型が `int const&` と決まった一つの参照だった。転送参照は、束縛するものに応じて参照の型そのものが変わる点が違う。

範囲for が `auto&&` を使うのは、名前付きの範囲でも一時オブジェクトでも、コピーせず、その値カテゴリのまま受けるためである。転送参照の詳しい規則には、本章では立ち入らない。
:::

型が `begin` と `end` をメンバ関数として持つときは、ここで見たように `__range.begin()` と `__range.end()` が使われる。`begin` と `end` は、メンバ関数のほか、名前空間の章で見た引数依存の名前探索で見つかる非メンバの関数として用意することもできる。

:::details 配列のイテレータ
配列を範囲for で回せるのは、配列でもイテレータの役目を果たすものがあるからである。配列の場合、それは要素を指すポインタである。先頭の要素を指すポインタがイテレータに、末尾の要素の次を指すポインタがセンチネルにあたる。ポインタは、指す先を隣の要素へ動かしたり、別のポインタと比べたりできる。その扱いそのものには基本編では立ち入らなかったが、これがイテレータとセンチネルの役目を果たす。配列では、この二つが同じポインタの型になる。
:::

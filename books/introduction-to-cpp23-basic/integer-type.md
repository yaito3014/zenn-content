---
title: "整数の型"
---

宣言の章では `int` を「型の名前」と呼び、関数の章では戻り値や仮引数の `int`、変数の章では変数の `int`、式と文の章では式の型としての `int` を述べた。いずれも詳しい意味には立ち入ってこなかった。本章ではこの `int` を題材に **整数の型** を扱う。

先に、本書での型の捉え方を示しておく。型は、値の種類を表す。種類ごとに、その値に対してどのような操作ができるかが決まる。

## `int` を見る

これまでに登場した `int` は、すべて整数を表す型を指している。整数リテラル `2`、`3`、`5` の型は `int` である。

`int` 型の値どうしには、たとえば次のような演算ができる。

- `+`：加算
- `-`：減算
- `*`：乗算
- `/`：除算(商)
- `%`：剰余

これらは代表的な演算であり、`int` 型に用意された演算のすべてではない。他の演算は必要になる章で扱う。

関数 `add` の本体に書かれた `a + b` は、`int` 型の値どうしの加算であり、結果も `int` 型の値となる。式と文の章で `a + b` の式の型が `int` であるとしたのは、これに基づく。

:::details 整数のふるまいは言語によって違う

整数の `/` やあふれの扱いは、言語によって異なる。

`/` について、C++ の `int` 同士の `/` は商(小数部を捨てた整数)になる。一方 Python では、`/` は常に小数の割り算を表し、整数同士でも結果は小数になる。

あふれについて、C++ の `int` には表せる範囲があり、その範囲を超えたときの符号付き整数の動作は規格上定められていない。一方 Python や Ruby では、整数が大きくなると自動的に多倍長の整数に切り替わり、あふれという概念がない。
:::

## 型はどこに現れるか

これまでに、型はいくつかの場所に現れた。

- 変数の型：`int result = add(2, 3);` の最初の `int`。
- 戻り値の型：`int add(int a, int b)` の最初の `int`。
- 仮引数の型：`int a`、`int b` の `int`。
- 式の型：`2`、`a + b`、`add(2, 3)` などの式が持つ型。

変数・関数の戻り値・関数の仮引数・式のいずれも、それぞれちょうど1つの型を持つ。

## 型はビルド時に検査される

C++ では、変数・式・関数の戻り値・仮引数の型は、すべてビルド時に定まる。書かれたコードの中で型の不整合があれば、ビルドツールはその時点で指摘する。

たとえば、`int` 型の仮引数を取る関数 `add` に文字列を渡そうとすると、ビルドに失敗する。

:::message
型の合わない値を渡す
```cpp
int result = add("two", "three");
```
`add` は `int` 型の値を2つ受け取るが、`"two"` や `"three"` は `int` 型の値ではない。ビルドに失敗する。
:::

このように、型の食い違いは実行を待たずビルドの段階で見つかる。型は、書いたコードが矛盾なく組み合わさっているかを、実行に先立って検証する仕組みでもある。

## `int` 以外の型

C++ には `int` 以外にもさまざまな型がある。真偽値を表す型や実数を表す型は、後の章で扱う。文字列を表す型は標準ライブラリが提供するもので、続きの編で扱う。

## 宣言の並びとの関係

プログラムを宣言の並びとみる見方は、ここでも変わらない。これまでに扱った範囲で型が現れる場所は、宣言の内部か、宣言の中に書かれた式の内部に限られる。

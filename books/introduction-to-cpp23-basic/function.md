宣言の章では、示したコードに宣言が3つあると述べ、そのうち2つを関数の宣言と呼んだ。本章ではこの関数の宣言を詳しく見ていく。

先に、本書での関数の捉え方を示しておく。関数とは、処理のまとまりに名前を与え、呼び出して動かせるようにしたものである。値を受け取ることもあれば、結果を返すこともある。値を受け取らない関数も、結果を返さない関数もある。

本書は、関数を「与えた値から結果を導く処理」として組む見方を主軸に据える。関数の外側に変化を残す働き(副作用と呼ばれる)を、関数の主な目的にはしない。`add` はその一例である。`a` と `b` を受け取り、`a + b` という値を導いて返すだけで、関数の外側には何も残さない。

とはいえ、画面への出力や入力の受け取りといった働きまで、こうした「値から値」の形に収めるための仕組みは、現状の標準ライブラリには用意されていない。そうした働きには、`std::println` のように外側への作用を担う関数を使う。

## `add` の宣言を分解する

宣言の章で示したコードのうち、`add` の宣言を再掲する。

```cpp
int add(int a, int b)
{
    return a + b;
}
```

この宣言は次の要素からなる。

- `int`：戻り値の型
- `add`：関数の名前
- `(int a, int b)`：仮引数の並び
- `{ return a + b; }`：関数の本体

順に見ていく。

### 戻り値の型と関数の名前

最初の `int` は、この関数が返す値の型である。値を返さない関数もあり、その場合の戻り値の型は `void` と書く。続く `add` は、この宣言で導入される名前であり、関数を呼び出すときに使う。

### 仮引数

`(int a, int b)` は仮引数の並びである。`int` 型の `a` と、`int` 型の `b` という2つの名前を、関数の本体の中で使えるようにする。

仮引数は、関数が呼び出されるたびにその都度値が与えられるものである。形は変数の宣言と似ているが、文法上は別の区分に置かれており、宣言の章で扱った宣言とは区別される。とはいえ、本体の中では `a + b` のように、ほかの名前と変わらず使える。

### 本体

`{ ... }` で囲まれた部分が本体である。本体には文が並ぶ。`add` の本体には1つの文があり、`return a + b;` がそれにあたる。

これは `return 文` と呼ばれ、関数の戻り値を決める文である。`return` のあとには式を書き、その式を評価した値が、関数を呼び出した側に戻る。`add` の場合、書かれた式は `a + b` であり、その値が `int` 型の戻り値として戻る。

## 関数を呼び出す

`add` の宣言を見たので、次は `add` を実際に使っている箇所を見る。`main` の本体にある次の式である。

```cpp
add(2, 3)
```

これは `add` を呼び出す式である。括弧の中に書いた `2` と `3` は、それぞれ仮引数 `a`、`b` に対応する値となる。これらは実引数と呼ばれる。

関数が呼び出されると、その関数の本体が実行される。`add` であれば `return` のあとの式 `a + b` が評価され、その値が呼び出し側に戻る。結果として、`add(2, 3)` は評価すると値 `5` になる式となる。

`main` ではこの値を `result` に結びつけている。

```cpp
int result = add(2, 3);
```

宣言の章で `result` の宣言を変数の宣言として挙げた。これがその宣言である。

## `main` の特別さ

宣言の章では「`main` は特別な関数である」とだけ述べた。本章でその特別さに踏み込む。

`main` も関数の1つであるが、次の点で他の関数と異なる扱いを受ける。

- プログラムを実行すると、`main` の本体から動き出す。
- 戻り値の型は `int` でなければならない。
- 本体の末尾に `return 文` を書かなくてよい。書かなかった場合は、本体の末尾に `return 0;` が書かれているのと同じ扱いになる。
- ユーザーが書いたコードから `main` を呼び出してはならない。

`main` は、プログラムの実行を始める入り口である。上の扱いの多くは、この役割に結びついている。

宣言の章で示した `main` には `return` がない。これは `main` だけに認められた省略である。`add` のような通常の関数で `return` を書き忘れると、ビルドツールは何も指摘しないことがあり、その関数を呼び出したときの結果が定まらず、実行時に深刻な問題を引き起こしうる。

:::message alert
戻り値を持つ関数で `return` を書き忘れる
```cpp
int twice(int x)
{
    int y = x + x;
    // ここで return を書き忘れている
}
```
こうした関数を呼び出した結果は定まらず、実行時に深刻な問題が起こりうる。戻り値を持つ関数で `return` を省略してよいのは `main` だけである。
:::

---
title: "コンストラクタとデストラクタ"
---

これまで、オブジェクトの初期化では、`Rectangle` のような型に `= {.width = 3, .height = 4};` のように `= {...}` で各データメンバを与えたり、`Counter` のようなクラスでデフォルトメンバ初期化子(`int count = 0;`)で初期値を決めたりしてきた。だが、オブジェクトを作るときに値を渡して、独自のしかたで初期化したい場合がある。それを担うのが **コンストラクタ** である。本章ではコンストラクタと、対になる **デストラクタ**(オブジェクトの寿命が尽きるときに呼ばれるもの)を扱う。

## コンストラクタ

アクセス制御の章の `Counter` は、`count` が必ず `0` から始まった。これを、作るときに好きな値から始められるようにしたい。`Counter` は `class` で `count` が private なので、`= {...}` では初期化できない。そこでコンストラクタを書く。

```cpp
class Counter
{
    int count = 0;

public:
    Counter(int start) : count(start)
    {
    }

    void increment()
    {
        count = count + 1;
    }

    int value() const
    {
        return count;
    }
};
```

`Counter(int start)` が、コンストラクタである。クラスと同じ名前を持ち、戻り値の型は書かない。オブジェクトを作るときに呼ばれ、ここでは `int` の引数 `start` を受け取る。

コンストラクタの仮引数の後に `:` を置き、続けて **メンバ初期化子リスト** を書く。ここでは `count(start)` がそれにあたり、データメンバ `count` を引数 `start` の値で初期化する。

作るときに、次のように値を渡す。

```cpp
Counter c(5);
std::println("{}", c.value());
```

`Counter c(5)` は、`Counter(int start)` を `start` が `5` で呼び出し、`count` を `5` で初期化する。出力は `5` になる。

## デフォルトコンストラクタ

ところが、コンストラクタを書いたことで、`Counter c;` のように引数なしで作る形がビルドに失敗するようになる。

:::message
引数なしでオブジェクトを作る
```cpp
Counter c;
```
`Counter` には `int` を1つ受け取るコンストラクタしかない。引数なしで作るコンストラクタがないため、ビルドに失敗する。
:::

アクセス制御の章で `Counter c;` と書けたのは、コンストラクタを1つも書いていなかったからである。いま `Counter(int start)` を書いたことで、引数なしで作る `Counter c;` は使えなくなった。

引数なしでも作れるようにするには、引数を取らないコンストラクタも書く。これを **デフォルトコンストラクタ** と呼ぶ。

```cpp
class Counter
{
    int count = 0;

public:
    Counter()
    {
    }

    Counter(int start) : count(start)
    {
    }

    void increment()
    {
        count = count + 1;
    }

    int value() const
    {
        return count;
    }
};
```

`Counter()` がデフォルトコンストラクタである。これで `Counter c;` も `Counter c(5);` も書ける。同じ名前で引数の違うコンストラクタを2つ持つ形で、これは型と関数の章で見たオーバーロードである。

ここで、メンバ初期化子リストとデフォルトメンバ初期化子の関係が見える。`Counter(int start)` はメンバ初期化子リストに `count` を書いているので、`count` は `start` で初期化される。一方 `Counter()` は `count` を書いていないので、`count` はデフォルトメンバ初期化子の `0` で初期化される。

つまり、メンバ初期化子リストに書いたメンバはその値で、書かなかったメンバはデフォルトメンバ初期化子の値で初期化される。どちらもなければ、そのメンバはデフォルト初期化され、`int` のような型では値が定まらない(デフォルト初期化の章で見たとおり)。

:::details メンバ初期化子リストと本体での代入
メンバ初期化子リストを使わず、コンストラクタの本体で `count = start;` と書くこともできる。ただしこの場合、`count` はいったんデフォルトメンバ初期化子で初期化され、そのあとに `start` が代入される。初期化と代入の違い(代入の章)が、ここにも現れる。メンバ初期化子リストはメンバを直接初期化するので、本書ではこちらを使う。
:::

## デストラクタ

コンストラクタがオブジェクトを作るときに呼ばれるのに対し、オブジェクトの寿命が尽きるときに呼ばれるのが **デストラクタ** である。クラス名の前に `~` を付けて書く。`Counter` に次のデストラクタを加える。

```cpp
~Counter()
{
    std::println("Counter を破棄しました");
}
```

`~Counter()` がデストラクタである。引数も戻り値の型も書かない。オブジェクトの寿命が尽きるときに、自動で呼ばれる。

```cpp
{
    Counter c(5);
}
```

`c` は、この `{ }` のブロックの中で作られ、ブロックを抜けるときに寿命が尽きる。そのときデストラクタが呼ばれ、`Counter を破棄しました` と出力される。ここでは、デストラクタが呼ばれることを確かめるために出力しているだけである。

デストラクタは、オブジェクトを使い終わるときに後始末をするためのものである。後始末が関わってくるのは、`new` と `delete` でオブジェクトを自分で作って壊すときである。それは後の章で扱う。本章では、コンストラクタと対になる仕組みとして、寿命の終わりに呼ばれることを押さえておく。

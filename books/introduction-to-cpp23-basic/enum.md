---
title: "列挙型"
---

基本型の章で見た型は、言語があらかじめ用意したものだった。C++ では、型を自分で定義することもできる。

状態や種類を表すのに整数を使うと、`0`・`1`・`2` がそれぞれ何を指すかは、数字だけからは分からない。代わりに、とりうる値に名前を付けた型を定義できる。これを **列挙型** と呼ぶ。本章では、現代的な `enum class` による列挙型を扱う。

## 列挙型を定義する

信号の色を表す型を考える。

```cpp
enum class Color
{
    red,
    green,
    blue
};
```

`enum class Color` は、`Color` という列挙型を定義する。`Color` がとりうる値は `red`・`green`・`blue` の3つで、それぞれの名前を **列挙子** と呼ぶ。

## 列挙型を使う

`Color` 型の値は、`型名::列挙子` の形で書く。

```cpp
Color c = Color::green;
```

`c` は `Color::green` を持つ。また、値が等しいかは `==` で調べられる。

```cpp
if (c == Color::red)
{
    std::println("赤");
}
```

`c` は `Color::green` なので、この条件は成り立たず、何も出力されない。

## 整数とは混ざらない

`enum class` で定義した列挙型の値は、整数とは別の型であり、整数へ暗黙には変換されない。

:::message
列挙型の値を整数として使う
```cpp
int n = Color::red;
```
`Color` の値は `int` に暗黙には変換されないため、ビルドに失敗する。
:::

名前を付けることで、値の意味がコードに現れる。整数と混ざらないことも、別の種類の値を取り違える誤りを防ぐ。

`class` を付けない `enum` もあるが、その値は整数へ暗黙に変換され、列挙子の名前も、型名を介さず周囲のスコープに置かれる。本書では、こうした混同を防ぐ `enum class` を使う。

:::details 他の言語の enum との違い
「enum」という名前は多くの言語にあるが、その値が何を持てるかは言語によって異なる。C++ の `enum class` の各値は、ひとつの整数に名前を付けたもので、それ以外のデータや操作は持たない。一方、言語によっては各値がより多くを持つ。Rust では、enum の各値(ヴァリアント)がデータを抱えられ、「値がある場合とない場合」のような区別を一つの型で表せる。Java では、各値がオブジェクトであり、フィールドやメソッドを持てる。同じ「enum」でも、C++ のそれは整数に名前を付けたものにとどまる。
:::

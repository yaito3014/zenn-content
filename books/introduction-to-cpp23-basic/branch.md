---
title: "条件分岐"
---

ここまでの章では、宣言の章で示したコードに現れた要素を順に分解してきた。宣言・関数・変数・式と文・整数の型・出力・プリプロセッシングディレクティブである。本章からは、そのコードには現れなかった要素に進む。最初に扱うのは **条件分岐** である。

先に、本書での条件分岐の捉え方を示しておく。条件分岐は、ある条件が成り立つかどうかによって、実行する文を選ぶしくみである。

## 例

2つの整数のうち大きいほうを返す関数 `max` を考える。

```cpp
int max(int a, int b)
{
    if (a > b)
    {
        return a;
    }
    else
    {
        return b;
    }
}
```

`max(2, 3)` は `3` を、`max(5, 1)` は `5` を返す。`a` と `b` のどちらが大きいかによって、返る値が変わる。これが条件分岐である。

## `if` 文

次の部分に注目する。

```cpp
if (a > b)
{
    return a;
}
```

これは `if` 文と呼ばれる。`if` に続く括弧の中に **条件** を書き、その条件が成り立つときに、続く `{ }` の中の文が実行される。`a > b` が成り立つ、すなわち `a` が `b` より大きいとき、`return a;` が実行される。

`if` 文も文の1つであり、式と文の章で見た文と同じく、関数の本体に並ぶ。

## 条件は真偽値の式

`a > b` は、これまで見てきた式と同じく、評価すると値になる式である。ただしその値は整数ではなく、「成り立つ」か「成り立たない」かのどちらかを表す値になる。この値は **真偽値** と呼ばれ、その型は `bool` である。整数の型の章で「真偽値を表す型」と呼んだのが、この `bool` にあたる。

真偽値を得る代表的な演算が、2つの値を比べる比較演算である。

- `a > b`：`a` が `b` より大きい
- `a < b`：`a` が `b` より小さい
- `a >= b`：`a` が `b` 以上
- `a <= b`：`a` が `b` 以下
- `a == b`：`a` と `b` が等しい
- `a != b`：`a` と `b` が等しくない

式と文の章で「すべての式は型を持つ」と述べた。`if` の条件として書く式は、`bool` 型の式である。

## `else`

`if` 文に続けて `else` を書くと、条件が成り立たなかったときに実行する文を指定できる。

```cpp
else
{
    return b;
}
```

`max` では、`a > b` が成り立たないとき、すなわち `a` が `b` 以下のとき、`else` に続く `return b;` が実行される。結果として、`max` はつねに大きいほうの値を返す。

条件が成り立つかどうかで、`return a;` と `return b;` のいずれか一方だけが実行される。両方が実行されることはない。

## 条件を括弧で囲む

`if` の条件は、括弧 `()` で囲まなければならない。囲み忘れると、どこからどこまでが条件かが定まらず、ビルドに失敗する。

:::message
`if` の条件を括弧で囲み忘れる
```cpp
if a > b
{
    return a;
}
```
条件 `a > b` が括弧で囲まれていないため、ビルドに失敗する。
:::

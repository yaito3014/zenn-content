---
title: "const"
---

変数の章で、本書は変数を「宣言と同時に初期化し、以後は書き換えない」形を主軸に据えると述べた。そのコラムでは、これは本書の方針であって、C++ では変数は既定で書き換えられる、と触れた。とはいえ、書き換えを言語に禁じさせる手段もある。本章でその手段、**`const`** を扱う。

## `const` をつける

変数の宣言に `const` を加えると、その変数を書き換えることは許されなくなる。

```cpp
int const x = 5;
```

`int` と `const` は、どちらを先に書いてもよい。`const int` と型の前に置いても、`int const` と同じ型を表す。順序は問われない。本書では、`const` がその左を修飾すると一貫して読めるよう、型の後ろに置く形に統一する。

では、この `x` を書き換えようとすると、どうなるか。

:::message
`const` をつけた変数に代入する
```cpp
int const x = 5;
x = 7;
```
`x` は `const` をつけて宣言されているため、2 行目の代入はビルドに失敗する。
:::

代入の章で見た書き換えを、`const` は宣言の時点で禁じる。書き換えてしまう誤りを、実行を待たずビルドの段階で止められる。

## `const` な変数は初期化しなければならない

`const` をつけた変数は、後から代入できない。値を結びつける機会は、宣言時の初期化しかない。

:::message
`const` な変数の初期化を省く
```cpp
int const x;
```
`x` は `const` なので後から代入できず、初期化子もない。値を結びつける方法がないため、ビルドに失敗する。
:::

## `const` は型に付く性質

`int const` は、`int` に「書き換えない」という性質を付した型である。`const` がその左の `int` を修飾し、この性質を加える。型が値の種類と操作を定めるように、`const` は「その値を書き換える操作を許さない」という制約を型に加える。`int const` 型の変数には、`int` 型の値を結びつけられるが、結びつけ直すことはできない。

:::details const と宣言指定子の並び

順序を入れ替えてよいのは、`const` に限った話ではない。`int const x = 5;` の `int const` の部分は、宣言の冒頭に置かれる **宣言指定子の並び**(`decl-specifier-seq`)であり、`int` も `const` も、この並びを構成する宣言指定子(`decl-specifier`)の一つである。並びを構成する指定子の順序は問われない。`int const` と `const int` が同じ型を表すのも、これによる。
:::

## 方針を言語に守らせる

変数の章では、「書き換えない」を本書の方針として述べた。`const` を使えば、この方針を言語に守らせることができる。書き換えないつもりの変数に `const` をつけておけば、誤って書き換えるコードを書いても、ビルドツールがそれを誤りとして指摘する。方針が、宣言の時点で保証に変わる。

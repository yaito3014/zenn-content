# 計画(章立て・ロードマップ)

この文書は **C++23入門シリーズ**の章立てと計画(roadmap)を扱う。執筆方針(公理・定理・規律=**政策**)は [guideline.md](guideline.md) にあり、ここで「方針」とあればそちらを指す。政策は安定し、計画は章の執筆につれて変動するため、ファイルを分けている。

書き上がった章の細かな起草判断は、**本文と git 履歴が出典**である。本文から復元できる記述は重複させず、各章には **回収関係**と**本文から復元しにくい非自明な判断**だけを一行で残す。

## 章立て(基本編)

基本編は部の区切りを設けない一続きの入門書。**前半**=アンカーコードを分解(アンカー駆動)、**後半**=言語機能をトピックごとに見る(トピック駆動)。標準ライブラリと扱いきれない言語機能は続きの編へ送り、おわりにで一区切り。**「一通り」「網羅」と読める書き方は避ける**。

背骨:宣言の並び(前半)→ 宣言が示す型・言語機能(後半)→ 型と関数の宣言の集まりであり `#include` で取り込む標準ライブラリ(続き)。

### 序
- **はじめに**(intro):序文(C++ 入門・C++23 基準・新機能網羅せず)、主題(ビルドが通ってしまう誤りを作法で防ぐ)、2種ボックスの約束、三層読者、旧スタイルは意図的に扱わない。
- **環境構築**(setup):cabin でビルド環境を作る。`cargo install cabinpkg` → `cabin new` → `cabin.toml` に `[profile] cxxflags=["-std=c++23"]`(既定は C++17)→ `src/main.cc` を `std::println` 版に差し替え → `cabin run`。公式手順への誘導中心・賞味期限の断り。cabin=ビルドツールを前面、コンパイラは背後(役割は翻訳章)。**バージョン依存の集約先**。手順は実機検証済み(cabin 0.14.0 / g++15 / Ninja)。

### 前半(アンカー駆動・確定)
- **宣言**(declaration):テーゼ「プログラムを宣言の並びとみる」。多くの宣言は名前を導入(`static_assert` 等で反証されるので断定しない)。モジュールは射程外=基本編では扱わない(コラム)。
- **関数**(function):関数=処理のまとまりに名前。値を返すのは本質でない(`void` も導入)。`main` の特別さ回収。`return` は式の値。副作用は標準ライブラリの都合として扱い「言語上できない」と書かない。
- **変数**(variable):変数=値に名前を結びつけたもの。宣言時に結びつけ以後書き換えない主軸(書き換えは代入章)。未初期化読み=赤。
- **式と文**(expression-and-statement):式と文の整理、アンカーの各要素を分類。`;` 忘れ=黄(初出)。型は別章。
- **整数の型**(integer-type):`int` の正体。3章の伏線回収。型はビルド時検査=黄。章名は scope を絞り「整数の型」。真偽値・実数の型は fundamental-types へ。
- **出力**(output):`std::println` の中身(書式 `{}`・`ln`・`#include <print>`・副作用)。`#include` 忘れ=黄。書式の詳細は続きの編。
- **プリプロセッシングディレクティブ**(preprocessing-directive):`#include` の中身。`<>` vs `""` は複数ファイル章へ(慣習は慣習と明示)。他ディレクティブ名の列挙・`#embed`(C++26)はコラム。基本編では立ち入らない。
- **条件分岐**(branch):if/else。比較演算子と `bool`(整数の型「真偽値を表す型」回収)。if の条件は `bool` 式。括弧忘れ=黄。
- **代入**(assignment):variable で送った書き換え。最大の精度=初期化と代入の区別。代入は式文。不変主軸を保ちつつ繰り返しの前提。ボックスなし。
- **繰り返し**(loop):while/for。書き換え前提なので代入の後。条件は `bool`、更新は代入。無限ループは散文注意(box なし)。for の `;` 厳密はコラム。
- **複数ファイル**(multiple-files):宣言と定義の区別を実地。ヘッダ(規格語、「ヘッダファイル」は使わない)、`""` vs `<>`(慣習)、翻訳単位とリンク。定義なし呼び出し=リンク失敗=黄。モジュールは使わずヘッダ+ソース。
- **翻訳**(translation):前半のキャップストーン。「ビルド」を一度開き preprocess→compile→link。ここで初めて「コンパイル」を本文導入(以降ビルドに戻す)。「コンパイラ」(道具)はコラムのみ。機械語は処理系の実態としてコラム。

### 後半(トピック駆動)
編成原理:型システムを背骨に。具体例から入り 2種ボックス・先送り・スタンス表明を守る。
- **基本型**(fundamental-types):bool/double/char を survey、`int` とともに基本型(受動形)。`double` の `/`(integer-type コラム回収)、符号付きあふれ=赤。文字列は続きの編。`void`/`nullptr` は関数・ポインタ章で回収。章名は規格 [basic.fundamental] に合わせる。
- **列挙型**(enum):`enum class`=とりうる値に名前。整数に暗黙変換されない=黄(型安全)。class なし `enum` は使わない(modern)。「列挙型=とりうる値に名前を付けた型」と定義してから結びつける。他言語比較コラム(Rust 直和/Java)。switch の前。
- **論理演算**(logical-operators):`&&`/`||`/`!`。基本型「bool の組み合わせは次章」回収。`1<=x<=10` を動機に。短絡評価。
- **その他の演算と複合代入**(more-operators):ビット演算・シフト(integer-type「他の演算」回収、survey+exclusion)、複合代入 `op=`・`++`/`--`(loop の `i=i+1` 回収)。前置/後置の式の値差を例示。box・まとめ節なし。
- **break と continue**(break-continue):繰り返しを抜ける/次へ。例は for。continue の精密(for の更新は走る/while は要注意)。範囲 for は続きの編。
- **switch**(switch):case 振り分け+enum でも(enum へ tie)。break 忘れ=流れ落ち(散文、UB でないので box なし)。
- **const**(const):型に「書き換えない」を付す。variable 回収。const 変数への代入=黄、初期化必須。east const 統一(前置も同義と一度断る)、文法裏づけはコラム。章名「const」(「定数」は規格の const 変数と一致しないので避けた)。
- **クラス**(class):自分で定義する型=規格語**クラス**(`struct`/`class`、本章は `struct`)。メンバ/データメンバ/メンバ関数、指示付き初期化 `{.x=…}` 主筋(位置指定は取り違え併記)、末尾 `;` 忘れ=黄。回収:基本型「型は値と操作を定める」。集成体・コンストラクタの語は本文に出さず details へ(`={…}` は集成体限定でないので hedge)。章名は規格語(『構造体』は規格の型カテゴリでない)。他言語比較コラム(C/Rust)。
- **デフォルト初期化**(default-initialization):初期値を書かない=デフォルト初期化。基本型は値定まらず、クラスもデフォルトメンバ初期化子なしなら同じ→未初期化メンバ読み=赤。デフォルトメンバ初期化子で安全。**default の訳し分け**:名前付き概念=デフォルト〜、一般の by default=既定。
- **アクセス制御**(access-control):public/private、struct/class の既定アクセス差(メンバの既定は本章/継承の既定は継承章)。private アクセス=黄。protected は一文 acknowledge し継承章へ。カプセル化は「設計の判断」(over-claim 回避)。例は Counter。
- **型と関数**(type-and-function):関数を型の観点で。関数の型 vs シグネチャ(戻り値型を含まない=通常関数、「含む」で非 exhaustive、変種はコラム)。値渡し・オーバーロード(戻り値型だけの違いは不可)・デフォルト引数。名前マングリングはコラム。参照渡し・ラムダ・再帰は続きへ送る。
- **参照とポインタ**(reference-and-pointer):参照=別名(束縛・再束縛不可)、初期化子なし=黄。参照渡し・const 参照(コピー回避)、const 越し書き換え=黄。ポインタ=アドレスを持つオブジェクト、`&`/`*`、指す先を変えられる、`nullptr`。null アクセス=赤。**binding 語は参照限定**。オブジェクト型の否定形は言い切らずコラム。
- **オブジェクト引数と const メンバ関数**(object-parameter):オブジェクト引数(`.` の左)を const より先に。const メンバ関数=書き換えない約束、const 参照に接続。const 内書き換え=黄。read-only な area/value を一斉 const 化(遡及回収)。**const correctness は訳さず英語呼称**。暗黙のオブジェクト引数はコラム。
- **コンストラクタとデストラクタ**(constructor-destructor):コンストラクタ=独自初期化。メンバ初期化子リスト(`:` を含まない)。デフォルトコンストラクタ:`Counter(int)` を書くと `Counter c;` が黄→引数なしを足す(暗黙生成の一般則は断定せず観察に閉じる)。デストラクタは寿命末に呼ばれる対概念。new/delete・RAII は new/delete 章へ。
- **型変換**(conversion):明示先行。`static_cast`、暗黙変換(=ビルド通る誤り、散文)、縮小変換+波括弧=黄、変換コンストラクタと `explicit`=黄。
- **継承**(inheritance):派生クラス。protected と struct/class の継承既定差(access-control 回収)。private→protected で派生から触れる。`: public` 明示を勧める。過度に推奨しない。多態は次章へ。
- **多態**(polymorphism):仮想関数・`override`・動的な型で呼び分け。**override 安全性=主題**(綴り間違い=黄、なしは silent bug)。スライシング=散文。仮想デストラクタはスタンス表明(why は new/delete へ)。「上書き/作り直す」は使わず**行為=オーバーライド・関係=優先される**(supersede)。
- **動的なオブジェクト**(new-delete):new/delete、基底ポインタ経由の破棄に仮想デストラクタが要る=赤。leak は散文(UB でない)。スマートポインタ(続きの編)へ橋渡し。
- **配列**(array):固定長/添字/範囲外=赤、動的配列 `new[]`/`delete[]`。`std::array`/`std::vector`(続きの編)へ橋渡し。生配列は推奨せず。範囲 for は続きの編。
- **auto**(auto):初期化子から型推論、静的に一つに決まる(動的型付けでない)を打ち消す。初期化子必須=黄。長い型名で有効。register は受動(「型は初期化子から決まる/推論される」)。
- **おわりに**(outro):基本編の振り返り。誤りを「ビルドに失敗するもの/ビルドが成功してしまうもの」の2種にボックスで区別した、と総括。扱わなかった言語機能・標準ライブラリは続きの編へ。

### 横断する設計判断(再議論を防ぐ)
- **制御構造を分岐と繰り返しに分けた理由**:while/for はループ変数の書き換えを要し、本書は不変主軸なので、書き換え(代入)導入後に繰り返しを置く。分岐は書き換え不要なので先。
- **explicit/const の遡及パターン**:導入前の章は素のまま据え置き、導入章が遡及ノートで一斉に回収する(片方だけの piecemeal にしない)。詳細は guideline §本書の content stance。
- テーゼへの繋ぎ直しは専用の章末節を立てず本文に織り込む(かつての全非宣言章の「宣言の並びとの関係」節は廃止し、translation のみ「構造と過程」節として残す)。詳細は guideline §編集の作法。

## 続きの編(別途/Zenn):言語編 + 領域別ライブラリ編

**編成原理:標準ライブラリは、あらかじめ用意された宣言の集まり。** 型(`std::vector`・`std::string` など)と関数(`std::println` など)の宣言をまとめたもので、`#include` で取り込んで使う。前半の宣言の見方・出力章の `std::println`・複数ファイル章のヘッダ取り込みと地続き。基本編で作った型と宣言の見方の土台の上に道具を載せる。手法(具体例から入る・2種ボックス・明示的な先送り・スタンス表明形・「網羅しない」)は据え置き。

**構成の決定(2026-06-04)**:続きは **言語編(①)→ 領域別のライブラリ編(②〜)**。言語機能(全ライブラリ編に共通の土台)を先に一編で据え、標準ライブラリは**領域ごとに編を分ける**(当初の「二編」から更新:ライブラリは広く領域ごとに性格が異なるため、一編でなく領域別の複数編へ)。依存順(土台→道具)を保ち、各編を「一区切りできる」単位にする(基本編の「部を作らず編で分ける」流儀と整合)。

**現代性の方針(2026-06-04)**:**modern を前面**に出す。エラー処理は `std::optional`/`std::expected`(C++23)を主役に、アルゴリズムは `std::ranges` を主軸にする。例外・反復子ペアは別系統の機構/補足として扱う。基本編の主題「ビルドの通る誤り・UB を作法で防ぐ/エラーを値として明示する」と噛み合うため。

**深度の方針(2026-06-04)**:公理・規律は基本編と同じだが、**概念をより深くインストールする**。読者像は基本編を終えた人(公理2の広さはその層に対して保つ)。深度の上げ方:(1) 基本編が opaque にした語(`std::`・`<T>`)を本体で正面から、(2) コラムへ逃がした厳密(集成体条件・シグネチャ変種・オブジェクト型)を必要な範囲で本体へ昇格、(3) UB は依然ヘッジ(公理1不変)だが「なぜ UB か」を寿命・無効化のモデルから説明する。

**深化した背骨**:基本編「宣言の並び → 型」を survey 深度で描いたのに対し、続きは同じ規律のまま2軸で深掘りする。(1) **総称化された宣言(テンプレート)**=標準ライブラリは総称の宣言の集まり(宣言/型テーゼの深化)。(2) **値とオブジェクトの素性(値カテゴリ・コピー/ムーブ・寿命)**=基本編がコピー・参照・new/delete で触れた所をモデルとして据える(コンテナ・スマートポインタの土台)。ライブラリ編はその payoff。

**土台の穴の回収**:基本編が opaque に使った言語要素(`void` は基本編 function 章で回収済み)のうち、続きで読者が書く側に回るもの=**名前空間**・**テンプレート**・**値カテゴリ/コピーとムーブ/寿命**を言語編の土台章にする。これらはコンテナ・アルゴリズム・スマートポインタの前提。

**言語編とライブラリ編の依存規律(2026-06-04)**:コア(②コンテナ・アルゴリズム等)の説明に要る言語機能(テンプレート・値カテゴリ/ムーブ・寿命・反復子・演算子オーバーロード・concepts・ラムダ)は、②だけでなく③④⑤にも共通する**横断的な土台**である。
- これらは**言語編に集約**し、各ライブラリ編に分散・重複させない(言語編を全ライブラリ編の前に置く根拠)。
- 循環(言語機能 X はライブラリ Y で動機づくが、Y は X を要る)は、**言語編では X を標準ライブラリ抜きの例で教える**ことで断つ(自前の型・組み込み型・手書き。例:範囲for は手製の iterable で)。ライブラリ編は X を std の型に当てて payoff を見せ、言語編へ参照を張る。
- **依存は言語編 → ライブラリ編の一方向**(言語編はライブラリ型に依存しない)。
- 特定の1ドメインだけに強く結びつき、そのライブラリ抜きでは動機づけられない機能だけ、その編に置く候補(大半は横断的なので原則は言語編集約)。

以下は候補で、順序・粒度・章名・要否は未確定。

### 続きの編①「言語編」(基本編外の言語機能を広く・深く)

**広さの方針(2026-06-04)**:基本編が扱わなかった言語機能を**なるべく広く**扱う。取り込みは**中程度まで**(核 + 高価値 + 中程度)。niche(ビットフィールド・RTTI 詳細・`volatile`)は除外。各機能は名前出しでなく深く据えるので分量は大きい(~20章規模)。「網羅した」と本文で主張する書き方は依然避ける。

依存順にテーマで束ねる(章の粒度・部の有無は草稿時に調整。基本編の「部を作らない」流儀が既定)。

**名前・宣言の発展**
- **名前空間(+ADL)**:定義・修飾/非修飾の名前探索・**ADL**(`std::cout << x`・`swap(a, b)` がなぜ動くか)。`std::` の正体を回収。基本編「ADL は深入りせず」を深化。
- **型エイリアス**:`using X = Y`。`typedef` は旧表記としてコラム。(alias template はテンプレート章へ:型エイリアス章はテンプレートより前なので素の `using` のみ)
- **ODR と inline**:基本編・複数ファイルの「宣言は複数回・定義は1つ」を **One Definition Rule** として据える。ヘッダ定義を複数 TU に取り込む多重定義=リンク失敗(黄)。`inline` で同一定義を複数 TU に置ける(ヘッダ/in-class 定義が可能に)。内部/外部リンケージ・無名名前空間は**コラムへ逃がす**(2026-06-04 決定)。
- **static メンバ**:`static` データメンバ(クラスに1つ・全オブジェクトで共有。`inline static` で in-class 定義=前章 ODR と inline を使う。クラス外定義はコラム)・`static` メンバ関数(`this` なし・`Class::f()`)。ODR/inline 章の後に置く(2026-06-04 分割)。
- **属性**:`[[nodiscard]]`・`[[maybe_unused]]`・`[[fallthrough]]` の**診断系**に絞る。戻り値の捨て忘れがビルドで警告=「防ぐ」主題に直結(nodiscard は既定で指摘=「指摘する」、fallthrough は `-Wextra` 等で=「指摘できる」と語を使い分け)。`[[fallthrough]]` は基本編 switch 章(意図しない流れ落ち)の回収。`[[no_unique_address]]` 等のレイアウト系は扱わない(メモリレイアウト章へ)。無視性の3規則 P2552R3(C++26)はコラム。

**総称**
- **テンプレート**:関数/クラステンプレート・**実引数推論**。`vector<int>` の `<int>` の正体。総称化された宣言=宣言テーゼの深化。alias template(`template<class T> using ...`)もここで(型エイリアス章から移動。テンプレート導入後に置く)。
- **オーバーロード解決**:基本編 type-and-function のオーバーロードの深化。候補→受け取れる候補→最良の候補(変換による順位)。受け取れる候補なし・曖昧=ビルド失敗(黄)。関数テンプレートも候補(本体が書けるかは問わず候補→実体化で失敗しうる=`twice` の例)。**concepts の前に置く根拠**:制約不満足の直接原因は「候補が外れ実行可能な候補が残らずオーバーロード解決が失敗」であって制約そのものでない(2026-06-04、concepts 起草中に判明)。concepts はこの上に「制約が候補の実行可能性を決める」を載せる。
- **concepts(制約)**:制約を満たさない型で候補が外れ、オーバーロード解決が失敗してビルドに失敗する(失敗の直接原因はオーバーロード解決=前章の上に載せる)。本体の奥でなく呼び出し側で、型への要求に即して失敗=主題と接続(深度上限の核)。**SFINAE はコラムで布石**:concepts 以前の機構(置換失敗はエラーにならない/`enable_if`・型特性)を一目示し、「既存・標準ライブラリのコードやエラーで見かける/concepts が現代の置き換え」と述べる。書く技法としては深入りせず、本体・独立章にはしない。
- **可変長テンプレート・特殊化**:パラメータパック(`println` の可変長の正体)・(部分)特殊化。テンプレートの発展。

**値とオブジェクトのモデル**(値モデルは3章=2026-06-04)
- **値カテゴリ**:lvalue/rvalue(必要なら xvalue/prvalue)。ムーブの土台。基本編に無い深い概念。コラムで lvalue/xvalue/prvalue の三分類と rvalue・glvalue のまとめを正しく示す(glvalue は lvalue の上位)。
- **= default と = delete**:特殊メンバ関数の有無を明示する道具。`= default`=自動生成版を使う(基本編 constructor-destructor のデフォルトコンストラクタ復活と接続)、`= delete`=関数を禁じる(どの関数にも使える)。**copy-and-move の前に独立章**(2026-06-04 決定):`= delete` が copy-and-move のムーブのみの型で初出だったのを解消、`= default` と対で導入。ただし copy/move 未導入の段で `= delete` を示すため、例はムーブのみの型でなく**オーバーロードの禁止**(`process(double)=delete`)とし、オーバーロード解決と対比(`= delete` は候補に残り選ばれてから不可=制約で候補が外れる concepts とは異なる)。「明示的にコンストラクタを宣言すると引数なしは**暗黙には宣言されない**」規則をこの章で初出として立てる(基本編 constructor-destructor は観察=`Counter c;` 失敗に留め一般則を断定していないので、既知から導かない)。`= default` は明示宣言+定義を自動生成にまかせる(成り立たなければ deleted)。**暗黙に delete される**節(参照メンバ等で default ctor が defaulted-as-deleted=候補に残り呼ぶと失敗、`= default` でも同じ)。宣言は暗黙/明示を毎回区別する。
- **コピーとムーブ**:特殊メンバ・`std::move`・ムーブのみの型(コピー `= delete`)。基本編のコピー(多態章スライシング)・値渡しの先へ。浅いコピー→二重解放=赤、コピー禁止→黄。`= default`/`= delete` は前章で導入済み。**defaulted な move が deleted なら候補から除外され copy が使われる**(規格 [class.copy] の除外規則)をコラムで(2026-06-05)。`= delete` した関数や deleted な default ctor が候補に残るのと対比。前章の「暗黙に delete される」を move に当てた payoff。
- **オブジェクトの寿命と記憶域**:自動/動的/静的の記憶域・寿命・**RAII の原理**。基本編 new/delete の深化。スマートポインタの土台。
- **メモリレイアウト**:オブジェクトの大きさ(`sizeof`)・空のメンバも場所を取ること・`[[no_unique_address]]`(C++20、空のメンバを詰める)。アラインメント/パディングは本体かコラムか草稿時に確定。記憶域=寿命の次に、記憶域=配置として置く(2026-06-04 新設。`[[no_unique_address]]` は属性章=診断系から分離)。

**型を作る発展**
- **演算子オーバーロード**:自分の型に `==`/`<`/`<<`/`+` 等。`operator<=>`(三方比較)・変換演算子も。値カテゴリ・コピー/ムーブと噛ませる。ADL の動機例(演算子)の回収。
- **friend(と hidden friend)**:`friend` で非メンバ関数や別クラスに private へのアクセスを許す(基本編アクセス制御章「friend は範囲外」の回収)。**hidden friend**=クラス本体内で定義する friend 関数で、**ADL でしか見つからない**(名前空間章 ADL の payoff)。名前空間を汚さず private に触れられるため、`==`/`<=>`/`<<` など対称な演算子を定義する現代的な定石。英語呼称 hidden friend を一度示す(定訳なし)。演算子オーバーロードの直後に置く(2026-06-04 決定。抽象クラス章から分離)。
- **ユーザー定義リテラル**:`operator""` で自分の型のリテラル(`1.0_km` 等)。演算子オーバーロードと対(演算子/リテラル構文を自分の型に広げる)。標準の `"..."s`/`"..."sv`・chrono の `1s`/`100ms` もこれ=ライブラリ編で使う側の布石。
- **反復子と範囲for**:範囲for は `begin`/`end`+反復子プロトコルの脱糖。自分の型に `begin`/`end` と最小の反復子(`operator*`/`++`/`!=`)を実装して範囲for を回す(標準ライブラリ不要)。基本編が「続きの編へ」と送った範囲for の deferral を回収。`begin`/`end` の発見=ADL(名前空間章)・反復子=演算子オーバーロードと接続。②の標準反復子・ranges の土台(分業:言語編=プロトコルを自分で作る/ライブラリ編=標準の反復子・ranges を使う)。
- **抽象クラスとインターフェース**:純粋仮想関数 `= 0`・多重/仮想継承(深入りは hedge)。基本編の継承・多態の発展。(friend は独立章へ分離)
- **他のキャスト**:`dynamic_cast`(多態と接続)・`const_cast`・`reinterpret_cast`。基本編 `static_cast` の続き。
- **構造化束縛**:`auto [a, b] = …`。tuple/pair(ライブラリ編)と接続。
- **共用体(union)**:軽く(`std::variant` はライブラリ編)。

**関数とコンパイル時**
- **ラムダ**:無名関数・キャプチャ(値/参照)・generic lambda。クロージャはオブジェクト、参照キャプチャの dangling=UB(寿命の章に接続)。②のアルゴリズムの布石。
- **再帰**:停止条件・呼び出しの積み重なり・止まらない再帰のスタック消費(ビルドは通るが実行時に問題)。
- **constexpr とコンパイル時計算**:`constexpr`/`consteval`/`constinit`。基本編で名前だけ出た `static_assert` と接続。
- **例外**:`try`/`catch`/`throw`(巻き戻し)・`noexcept`。主役には据えず、値ベースのエラー処理(②)と別系統と位置づける。例外の置き場(①/②)は草稿時に確定。

### 続きの編②〜:領域別ライブラリ編(道具・宣言の集まり・modern 前面)
標準ライブラリは広いので領域ごとに編を分ける。各編は言語編の上に乗る(上記の依存規律)。順序・粒度・編の境界は未確定。

**② コンテナと値の型編**(旧「コンテナとアルゴリズム編」。2026-06-09 決定:③ヴォキャブラリ型を統合。②が短いとの判断で、コンテナ/アルゴリズムに加え、一つの値を表す型 optional/expected/variant/any を同じ巻に収めた。タイトルは「コンテナと値の型編」。後半=値の型として下記④群を本巻に移設。④入出力・⑤時間数値は引き続き別編)
- **文字列**:`std::string`・`std::string_view`(非所有ビュー→言語編の寿命に接続)。integer-type の「文字列を表す型」の最終回収先。
- **シーケンスコンテナ**(章名カタカナ、旧「列コンテナ」):`std::vector`・`std::array`・`std::deque`(2026-06-09 追加:両端に足せる=push_front/push_back、vector の末尾のみと対比、本節)。配列章からの橋渡し。標準のイテレータ(言語編「反復子と範囲for」の実例)・**無効化**(成長/削除でムーブ・寿命に接続)。範囲for の脱糖は言語編で済み。値の並びは「並び」、鍵→キー。
- **連想コンテナ**:`std::map`・`std::set`、および `std::unordered_map`・`std::unordered_set`(2026-06-09 決定:本節として正面から扱う)。順序(map/set)対 ハッシュ・非順序(unordered)の対比を軸に、ハッシュ・hashable なキー(用意なしはビルド失敗=黄)・規格の連想/非順序連想の分類まで。性能は複雑度の語彙を使わず機構でヘッジ。さらに `std::flat_map`・`std::flat_set`(C++23)も本節として(2026-06-09 決定):map/set と同じく順序を保つが要素を連続して持つ、コンテナアダプタ、連続格納のトレードオフ(挿入/削除で後ろがずれる)まで。**flat_map は SoA**(キー配列+値配列、pair の AoS でない=規格 [flat.map] の二コンテナ格納)で、SoA/AoS はコラム。連想コンテナは計6つ(map/set・unordered・flat)。
- **ranges とアルゴリズム**:`std::ranges` を主軸。言語編のラムダを使う。反復子ペアはコラム/補足。projection は本節として扱う(2026-06-09 決定):要素の一部分で sort/find/count、`{}`=既定比較、ラムダ projection と `&Class::member`(メンバを指すもの、として軽く)。projection の語は英語呼称(定訳なし)。
- **ビューと複合**:`std::span`・`std::tuple`/`std::pair`(構造化束縛=言語編 の payoff)。span は extent も扱う(2026-06-09 決定、span 節に組み込み):動的 extent(`span<int>`=実行時の長さ)対 静的 extent(`span<int,N>`=型に固定)、size 例、長さ不一致はビルド失敗=黄。extent は英語呼称(定訳なし)。
- **スマートポインタ**:`std::unique_ptr`/`std::shared_ptr`。所有権=言語編のムーブ+寿命の payoff。new/delete 章の回収先。**②の末尾に置く**(2026-06-08 決定):所有権の payoff を締めに据え、vector の無効化(ムーブ・寿命)と同じ巻で並べる。「冒頭の資源を持つ型/別置き」案は不採用。

**②後半 値を表す型**(旧③ヴォキャブラリ型編。2026-06-09 に②へ統合、各型1章=4章)
- `std::optional`(値があるかないか。空の取り出し=UB は赤、value() は bad_optional_access)。**monadic operations**(transform/and_then/or_else、C++23)も本節で(2026-06-09 追加):if なしでつなぎ空で短絡。monad 理論は説明せず英語呼称のみ。
- `std::expected`(C++23、エラー処理の主役。基本編の主題「誤りを作法で防ぐ/エラーを値として明示」に直結。言語編の例外と対比=別系統)。monadic operations も(optional 節を受けて簡潔に、エラーが伝わる版)。
- `std::variant`(言語編の union と対比=**閉じた**型集合の直和。union と違い「今どの型か」を追う)
- `std::any`(**型消去**で任意の型を1つ。variant の閉じた集合に対し**開いた**型、という対比)
- 章順:optional→expected→variant→any。スマートポインタの後、おわりにの前。「ヴォキャブラリ型」の語は本文に出さず、章は「値があるかないか」等の概念で。

**④ 入出力編**
- 出力の書式:`std::println`/`std::format`(output.md の deferral 回収)
- 入力:`std::cin`・ストリームの状態・パース
- ファイルと filesystem:`std::ifstream`/`std::filesystem`

**⑤ 時間・数値・ユーティリティ編**
- 時間:`std::chrono`(UDL `1s`/`100ms` で言語編と接続)
- 乱数:`<random>`
- 数値・数学:`<numeric>`/`<cmath>`
- ビット操作:`std::bit`

**⑥ 並行編(上級・後置)— 当面 exclusion(2026-06-08 決定)**
- スレッド(`std::thread`/`std::jthread`)・同期(`std::mutex` 等)・アトミック・非同期(`std::future`/`std::async`)。重く上級なので最後・別編。**今回の射程には入れない**:②〜⑤を完成させることに集中し、並行は「この先で扱う」と明示的に先送りする(宙吊りにせず exclusion として書く)。後から起こす。

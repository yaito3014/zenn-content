本書は、これから C++ を書く人に向けた入門書である。説明とコード例は C++23 を基準とし、C++23 以降に対応した環境を前提とする。C++23 で新しく加わった機能を網羅することは目的とせず、いま C++ を新しく書くときの書き方に沿って解説する。

C++ という言語において頻繁に現れる現象として、「誤ったコードに対しビルドが成功してしまう」ことが挙げられる。
ビルドを伴う言語としては「誤ったコードに対してはビルドが成功しない」ことが望ましいが、様々な理由から、その限りではないことがある。
そしてビルドが成功してしまう誤りの一部は、実行時に深刻な問題を引き起こしうる。
本書が主題とするのは、こうした「ビルドツールが指摘しない誤り」を、書き方の作法と道具によって未然に防ぐことである。そのため本書では、二種類の誤ったコードを色で区別して示す。

ビルドが成功しない誤ったコード、すなわちビルドツールが指摘する誤りは、以下のように黄色く示す。

:::message
ビルドが成功しない誤ったコード
:::

ビルドが成功してしまう誤ったコード、とりわけ実行時に深刻な問題を引き起こしうる誤りは、以下のように赤く示す。ビルドツールは何も指摘しないことがあるため、本書ではこの区別をとくに重視する。

:::message alert
ビルドが成功してしまう誤ったコード
:::

本書では、短い完成済みのプログラムを出発点に、その構成要素を章ごとに分解して見ていく。まだ説明していない要素については、どこで扱うかを明示したうえで、その場では立ち入らない。

## 対象読者

本書は、次のいずれの読者も対象とする。プログラミングを初めて学ぶ人、他の言語をすでに習得している人、そして以前に C++ を学んだことのある人である。

初めて学ぶ人は、本文を順に読み進められる。前提となる知識は特に求めない。

他の言語の経験がある人は、各所に設けたコラムで、そうした言語と C++ との差異を確認できる。慣れ親しんだ言語では当然とされていた事柄が、C++ では異なる形をとることがある。

以前に C++ を学んだことのある人には、一点ことわっておく。本書は C++ を「いま新しく書くならどう書くか」という観点で解説するため、かつて標準的とされた書き方を意図的に扱わない箇所がある。見慣れた書き方が現れないとしても、それは欠落ではなく、本書の方針による。

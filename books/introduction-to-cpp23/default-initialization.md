---
title: "デフォルト初期化"
---

変数を作るとき、`int x = 5;` のように `=` のあとに初期値を書いてきた。クラスのオブジェクトも、`Rectangle r = {.width = 3, .height = 4};` のように初期値を書いて作った。初期値を書かずに作ることもでき、これを **デフォルト初期化** という。本章でこれを扱う。

## 初期値を書かないと

変数の章で、`int result;` のように初期化を省くと値が定まらない、と述べた。これが基本型のデフォルト初期化である。`int` のような基本型では、デフォルト初期化をした変数の値は定まらない。

クラスのオブジェクトも、初期値を書かずに作れる。クラスの章の `Rectangle` で見る。

```cpp
Rectangle r;
```

`Rectangle` のデータメンバ `width` と `height` には、初期値を書いていなかった。そのため、`Rectangle r;` とすると、`width` と `height` の値は定まらない。`int result;` と同じである。

:::message alert
デフォルト初期化で値の定まらないメンバを読む
```cpp
Rectangle r;
std::println("{}", r.width);
```
`width` に初期値がないため、`Rectangle r;` とした `r` の `width` は定まらない。これを読み取った結果は定まらず、実行時に深刻な問題が起こりうる。
:::

## デフォルトメンバ初期化子

データメンバに初期値を書いておけば、デフォルト初期化でもその値になる。データメンバに与えるこの初期値を **デフォルトメンバ初期化子** と呼ぶ。

```cpp
struct Rectangle
{
    int width = 0;
    int height = 0;

    int area()
    {
        return width * height;
    }
};
```

`width` と `height` に `= 0` と書いておくと、`Rectangle r;` とした `r` の `width` と `height` は `0` になる。値が定まらない心配がなくなる。

## 独自の初期化

型は、初期化のしかたを自分で定めることもできる。それを担うのが **コンストラクタ** であり、コンストラクタを定めた型は、初期値の書き方や、初期化のされ方が変わる。本書では後の章で扱う。

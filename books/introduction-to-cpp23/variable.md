---
title: "変数"
---

宣言の章では、`result` の宣言を変数の宣言と呼んだ。関数の章では、`int result = add(2, 3);` を、値を `result` に「結びつけている」と曖昧に述べた。本章ではその `result` を詳しく見て、変数とは何かを扱う。

先に、本書での変数の捉え方を示しておく。変数とは、ある値に一つの名前を結びつけ、後でその名前から値を取り出せるようにしたものである。関数が「処理のまとまりに名前を与えたもの」だったのに対し、変数は「値に名前を結びつけたもの」と捉える。

本書は、変数を「宣言時に値を一つ結びつけ、以後はその値を参照できるようにしたもの」として扱う見方を主軸に据える。C++ では変数の値を後から書き換えることもできるが、できるだけ初期化のみで完結する形を中心に置く。書き換えが要る場面は、必要になったときに扱う。

## `result` の宣言を分解する

宣言の章で示したコードのうち、`result` の宣言を再掲する。

```cpp
int result = add(2, 3);
```

この宣言は次の要素からなる。

- `int`：変数の型
- `result`：変数の名前
- `= add(2, 3)`：初期化子

順に見ていく。

### 型

`int` は、この変数が保持する値の型である。型は、変数に結びつけられる値の種類を定めるもので、すべての変数はちょうど一つの型を持つ。`int` は整数を表す型である。

これまでに出てきた `int` は、すべてこの「型」を指していた。関数 `add` の戻り値の型として `int`、仮引数 `a` と `b` の型として `int`、本章の変数 `result` の型として `int`。型については別の章で改めて扱う。

### 変数の名前

`result` は、この宣言で導入される名前であり、変数を参照するときに使う。関数の章で `add` という名前を関数の呼び出しに使ったのと同様、変数の名前は、結びついた値を取り出すときに使う。

### 初期化子

`= add(2, 3)` は初期化子と呼ばれる。`=` の後ろには式を書き、その式を評価した値を、変数に結びつける。これを初期化と呼ぶ。

関数の章で、`add(2, 3)` は値 `5` をもつ式だと述べた。したがって、この宣言は値 `5` を `result` という名前に結びつけることを表す。関数の章で「結びつけている」と曖昧に述べたのは、この初期化を指していた。

なお、本章で扱った `int result = add(2, 3);` は、変数の宣言であり、実体まで与えているため、宣言の章で述べた **定義** にも当たる。

## 変数を使う

初期化した変数は、その名前を書くことで結びつけた値を取り出せる。`main` の本体にある次の文を見る。

```cpp
std::println("{}", result);
```

ここでは `result` という名前を使い、結びついた値、すなわち `5` を取り出して `std::println` に渡している。`std::println` の働きの詳細は別の章で扱うが、ここでは `result` の値を画面に出力するために渡している、と読めればよい。

仮引数を関数の本体の中で `a + b` のように使ったのと、使い方は同じである。仮引数は関数が呼ばれるたびに値が結びつくものであり、変数は宣言時に値を結びつけて以後参照されるものである。両者は文法上は別の区分だが、名前から結びついた値を取り出すという点で似た働きを持つ。

## 未初期化の変数を読み取らない

C++ では、変数の宣言時に初期化を省くこともできる。

```cpp
int result;
```

ただし、初期化を省くと、この変数には特定の値が結びつかない。この状態の変数を読み取ろうとすると、結果は定まらず、実行時に深刻な問題を引き起こす。

:::message alert
未初期化の変数を読み取る
```cpp
int result;
std::println("{}", result);
```
`result` には特定の値が結びついていない。これを `std::println` に渡したときに何が出力されるかは定まらず、実行時に深刻な問題が起こりうる。
:::

本書は、関数の章で副作用を主目的にしない方針を据えたのと同様、変数についても「宣言と同時に初期化する」を基本に据える。

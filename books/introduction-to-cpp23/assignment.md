---
title: "代入"
---

変数の章では、変数を「宣言時に値を1つ結びつけ、以後その値を書き換えない」形を主軸に据えると述べ、書き換えが要る場面には立ち入らなかった。本章でその書き換え、すなわち **代入** を扱う。

先に、本書での代入の捉え方を示しておく。代入は、すでに宣言された変数に、新しい値を結びつけ直す操作である。

## 代入の例

次のコードを `main` の本体に書いたとする。

```cpp
int x = 5;
std::println("{}", x);
x = 7;
std::println("{}", x);
```

これを実行すると、`5` と `7` がこの順に出力される。1つ目の出力の時点では `x` に `5` が結びついている。続く `x = 7;` によって `x` の値が `7` に書き換わるため、2つ目の出力は `7` になる。この `x = 7;` が代入である。

## 初期化と代入は違う

ここで、よく似た2つの行を見比べる。

```cpp
int x = 5;
x = 7;
```

どちらも `=` を使っているが、両者は別の操作である。

1行目の `int x = 5;` は変数の宣言であり、`= 5` は変数の章で見た初期化子である。変数を宣言すると同時に、最初の値を結びつける。これは **初期化** と呼ばれる。

2行目の `x = 7;` は、すでに宣言された変数 `x` の値を新しい値に結びつけ直す。これが **代入** である。ここでの `=` は代入を表す演算子であり、初期化子の `=` とは別のものである。

見た目は同じ `=` でも、初期化は宣言の一部であり、代入はそれ自体が1つの操作である。前者は変数に最初の値を結びつけ、後者はすでに結びついた値を別の値に置き換える。

## 代入は式文

`x = 7` は、変数 `x` に値を結びつけ直すことを表す式である。これに `;` を付けた `x = 7;` は、式と文の章で見た式文にあたる。代入は、関数の本体に並ぶ文の1つとして実行される。

## 本書での書き換えの扱い

変数を書き換えなければ、変数の値はその宣言を見れば分かる。代入によって書き換えを許すと、変数の値は実行の流れの中で移り変わるため、どの時点でどの値であるかは宣言だけからは分からなくなる。

本書は、この理由から、変数を書き換えない形を引き続き主軸に据える。とはいえ、書き換えが自然にあてはまる場面もある。次の章で扱う繰り返しがその一例であり、そこでは代入が中心的な役割を担う。

## 宣言の並びとの関係

プログラムを宣言の並びとみる見方は、ここでも変わらない。代入は、関数の本体に並ぶ式文として現れるものであって、宣言の並びという枠組みそのものを変えるわけではない。

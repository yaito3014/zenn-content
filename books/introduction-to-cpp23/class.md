title: "クラス"
---

基本型の章で見た型は、すべて言語があらかじめ用意したものだった。C++ では、複数の値をまとめた新しい型を、自分で定義できる。`struct` や `class` というキーワードで定義する型は、**クラス** と呼ばれる。この2つのキーワードには違いがあるが、それは後の章で扱う。本章では `struct` を使う。

## クラスを定義する

長方形を表す型を考える。幅と高さを、1 つの型にまとめたい。

```cpp
struct Rectangle
{
    int width;
    int height;
};
```

これは `Rectangle` という新しい型を定義する。`width` と `height` は、この型の **メンバ** である。とくに、データを保持するメンバを **データメンバ** という(メンバ変数とも呼ばれる)。`width` と `height` は、どちらも `int` 型のデータメンバである。

宣言の章で、多くの宣言は名前をプログラムに導入すると述べた。そこで導入された名前は、値や関数を指すものだった。クラスの定義は、新しい **型** を導入する宣言である。

クラスの定義は、閉じ括弧 `}` のあとの `;` で終わる。この `;` を書き忘れると、ビルドに失敗する。

:::message
クラスの定義の後の `;` を書き忘れる
```cpp
struct Rectangle
{
    int width;
    int height;
}
```
`}` のあとに `;` がないため、ビルドに失敗する。
:::

## クラスを使う

定義した `Rectangle` 型は、`int` などと同じく、変数の型として使える。

```cpp
Rectangle r = {.width = 3, .height = 4};
```

変数と同じく `=` で初期化する。`{}` の中に、`.メンバ名 = 値` の形で各データメンバの値を指定する。どの値がどのデータメンバになるかが、書いた時点で分かる。この書き方は **指示付き初期化** と呼ばれる。

データメンバの名前を省き、値だけを並べることもできる。

```cpp
Rectangle r = {3, 4};
```

この場合、値は宣言した順に対応し、`width` が `3`、`height` が `4` になる。`{4, 3}` と書いても誤りにはならず、意図とは違う値が入りうる。名前で指定しておけば、こうした取り違えを防げる。本書では、指示付き初期化を主に使う。

なお、`= {...}` という波括弧での初期化が使えるか、また何を意味するかは、型の定め方による。`Rectangle` のような型では、ここで見たようにデータメンバを初期化する。型によっては別の意味になることもある。

:::details 集成体とリスト初期化

`Rectangle` のような型は **集成体**(aggregate)と呼ばれる。集成体になるにはいくつかの条件があり、たとえばユーザーが宣言したコンストラクタを持たないこと、データメンバがすべて公開されていることなどである。正確な条件は規格に定められ、版を追って変わってきた。仮想でないメンバ関数を持っていても、集成体であることは妨げられない。

`= {...}` は **リスト初期化** と呼ばれ、何をするかは型によって決まる。集成体に対しては、メンバを初期化する **集成体初期化** になる。本章で見た位置指定も指示付き初期化も、集成体初期化の書き方である(指示付き初期化は集成体に対してのみ使える)。一方、コンストラクタを持つ型に対しては、適合するコンストラクタが呼ばれる。たとえば `std::vector<int> v = {1, 2, 3};` は、`std::initializer_list` を取るコンストラクタを呼ぶ。

:::

データメンバには `.` でアクセスできる。

```cpp
std::println("{}", r.width);
```

これは `3` と出力する。

## メンバ関数

クラスは、`width` や `height` のようなデータメンバだけでなく、関数も持てる。型に属するこの関数も、メンバの一つで、**メンバ関数** と呼ばれる。

`Rectangle` に、面積を求めるメンバ関数を加える。

```cpp
struct Rectangle
{
    int width;
    int height;

    int area()
    {
        return width * height;
    }
};
```

`area` は、その `Rectangle` のデータメンバ `width` と `height` を使って面積を返す。データメンバに `.` でアクセスしたように、メンバ関数も `.` で呼び出す。

```cpp
Rectangle r = {.width = 3, .height = 4};
std::println("{}", r.area());
```

これは `12` と出力する。`r.area()` は、`r` のデータメンバ `width`、`height` を使って計算する。

:::details C や Rust の構造体との違い
データをまとめた型に操作(関数)をどう結びつけるかは、言語によって異なる。C++ では、クラスの定義の中にデータメンバとメンバ関数の両方を書く。C の構造体(`struct`)はデータをまとめるだけで、関数は持てない(関数は構造体の外に書く)。同じ `struct` でも、C++ では関数まで持てる。また、C のデータだけの構造体はそのまま C++ で使え、同じメモリ配置で両言語の間を受け渡せる。ただし、C++ 独自の機能を加えると配置が変わることがあり、互換は部分的にとどまる。Rust の構造体(`struct`)はデータ(フィールド)をまとめ、操作はそれとは別の `impl` ブロックに書く。データと操作をどこに置くかが、言語ごとに異なる。
:::

## 型を定義するとは

基本型の章で、型は値の種類と、その値にできる操作を定める、と述べた。クラスを定義することは、これを自分で行うことにほかならない。`width` や `height` のようなデータメンバでどんな値を持つかを決め、メンバ関数でどんな操作ができるかを決める。`int` のような型が言語から与えられたのに対し、`Rectangle` のような型は自分で導入する。

なお、クラスのメンバには、どこからでも自由に触れてよいものと、隠しておきたいものとがある。この区別(アクセス制御)は、後の章で扱う。`struct` と `class` の違いも、そこで扱う。

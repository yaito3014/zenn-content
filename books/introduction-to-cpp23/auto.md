---
title: "auto"
---

本書はここまで、変数の型を `int`・`double`・`Rectangle` のように明示して書いてきた。型が初期化子から明らかなときは、型を書く代わりに **`auto`** と書ける。型は初期化子から決まる。

## auto で型を推論する

```cpp
auto n = 5;
auto d = 3.14;
```

`auto n = 5;` では、初期化子 `5` が `int` なので、`n` の型は `int` になる。`auto d = 3.14;` では、`d` は `double` になる。このように、初期化子の型から変数の型を決めることを **型推論** と呼ぶ。

型は、ビルド時に一つに決まる。一度決まれば、`int` や `double` と明示して書いたときと変わらず、後から別の型に変わることはない。`auto` は型を書く手間を省くものであって、どんな型でも入れられるようにするものではない。

## 初期化子が要る

`auto` は初期化子の型から推論するので、初期化子がないと型を決められない。

:::message
auto に初期化子がない
```cpp
auto n;
```
`n` の型を決める手がかり(初期化子)がないため、ビルドに失敗する。
:::

## 型の名前が長いとき

型の名前が長いほど、`auto` は書く量を減らす。動的なオブジェクトの章で `Counter* p = new Counter();` と書いたが、これは次のように書ける。

```cpp
auto p = new Counter();
```

`new Counter()` は `Counter*` を返すので、`p` の型は `Counter*` になる。`Counter*` と書く代わりに `auto` で済む。

明示した型と `auto` は、同じ一つの型を表す。型そのものに目を向けたいときは型を書き、初期化子から型が明らかなときは `auto` を使う。

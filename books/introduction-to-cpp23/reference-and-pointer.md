---
title: "参照とポインタ"
---

型と関数の章で、値渡しでは実引数の値がコピーされ、関数から呼び出し側のオブジェクトを変えられない、と述べた。また、大きなオブジェクトを渡すと、その分のコピーが生じる。どちらも、オブジェクトそのものを指す **参照** で扱える。本章では参照と、それに関連する **ポインタ** を見る。

先に、本書での捉え方を示しておく。参照は、既存のオブジェクトに付ける別名である。変数が値に名前を結びつけたものだったのに対し、参照は、すでにあるオブジェクトにもう一つ名前を与える。`int& r = x;` は新しいオブジェクトを作らない。`int` のオブジェクトは `x` 一つのままで、`r` はそれに与えたもう一つの名前にすぎない。

## 参照は別名

`int& r = x;` で参照を作る。

```cpp
int x = 5;
int& r = x;
```

`r` は `x` の別名である。`r` と `x` は同じ一つのオブジェクトを指す。だから `r` を通して書き換えると、`x` も変わる。

```cpp
r = 10;
std::println("{}", x);
```

`r = 10;` は、`r` が指すオブジェクト、すなわち `x` に `10` を書き込む。出力は `10` になる。

変数 `int y = x;` と比べる。これは `x` の値をコピーした新しいオブジェクト `y` を作り、`y` を書き換えても `x` は変わらない。一方 `int& r = x;` は、新しいオブジェクトを作らない。オブジェクトは `x` 一つで、名前が `x` と `r` の二つになる。

参照それ自体はオブジェクトではない。`r` という名前は `x` を指すだけで、`r` のための新しいオブジェクトがどこかにできるわけではない。

:::details 参照はオブジェクトではない
規格には **オブジェクト型** という型の分類があり、参照型(`int&` など)はそこに含まれない。参照が記憶域を占めるかどうかも、規格は規定していない。後で見るポインタはオブジェクト型であり、参照とはこの点で異なる。
:::

参照は、作るときに必ず結びつける相手が要る。参照を対象に結びつけることを **束縛** と呼ぶ。結びつけずに参照だけを作ることはできない。

:::message
結びつける相手なしに参照を作る
```cpp
int& r;
```
`r` を何に結びつけるかがないため、ビルドに失敗する。
:::

いったん束縛した参照は、別のオブジェクトに再束縛できない。`r = y;` と書いても、`r` が `y` を指すようにはならず、`r` が指すオブジェクト(`x`)に `y` の値が書き込まれるだけである。

## 参照渡し

仮引数を参照にすると、関数は実引数のオブジェクトそのものを受け取る。型と関数の章で見た、値渡しの `add_one` と比べる。

```cpp
void add_one(int& n)
{
    n = n + 1;
}
```

```cpp
int x = 5;
add_one(x);
std::println("{}", x);
```

仮引数 `n` は `int&`、すなわち参照である。`add_one(x)` と呼ぶと、`n` は実引数 `x` の別名になる。だから `n = n + 1;` は `x` を書き換える。出力は `6` になる。値渡しのときに `x` が `5` のままだったのと対照的である。これを **参照渡し** と呼ぶ。

ただし本書は、関数の章で述べたとおり、外側のオブジェクトを書き換えるより値を返す形を主軸にする。参照渡しは、呼び出し側のオブジェクトそのものを指す必要があるときの手段として使う。

## const 参照

参照を通した書き換えを禁じるには、参照を const にする。

```cpp
void print_size(Rectangle const& r)
{
    std::println("{} x {}", r.width, r.height);
}
```

`Rectangle const& r` は、`Rectangle` への const な参照である。`r` を通して `Rectangle` のデータメンバを読めるが、書き換えはできない。

```cpp
Rectangle box = {.width = 3, .height = 4};
print_size(box);
```

参照渡しなので、`print_size(box)` と呼んでも `Rectangle` のコピーは作られない。`const` を付けてあるので、関数の中で誤って書き換えることもない。オブジェクトが大きいほど、コピーを避ける効果は大きい。型と関数の章で「大きなオブジェクトのコピーを避けたい」と述べたのは、この const 参照で扱える。

:::message
const 参照を通して書き換える
```cpp
void print_size(Rectangle const& r)
{
    r.width = 0;
}
```
`r` は const な参照なので、通して書き換えることはできない。ビルドに失敗する。
:::

本書では、コピーを避けて読み取るための const 参照を、参照の主な使い道とする。

## ポインタ

参照と並んで、他のオブジェクトを指すもう一つの仕組みに **ポインタ** がある。ポインタは、指す先のオブジェクトの **アドレス**(オブジェクトのある場所)を値として持つ。

```cpp
int x = 5;
int* p = &x;
```

`&x` は `x` のアドレスを表す。`int* p` の `int*` は `int` を指すポインタの型を表し、`p` に `x` のアドレスを入れている。`p` を通して `x` にアクセスするには、`*p` と書く。

```cpp
*p = 10;
std::println("{}", x);
```

`*p` は `p` が指すオブジェクト、すなわち `x` を表す。`*p = 10;` で `x` は `10` になる。

ポインタは参照と違い、それ自体が一つのオブジェクトである。アドレスを値として持ち、後から別のオブジェクトのアドレスに入れ替えられる。

```cpp
int y = 20;
p = &y;
```

これで `p` は `y` を指す。参照が再束縛できなかったのに対し、ポインタは指す先を変えられる。また、ポインタは何も指さない状態も表せる。これを `nullptr` で表す。

```cpp
int* p = nullptr;
```

何も指していないポインタを通してアクセスしようとすると、結果は定まらず、実行時に深刻な問題が起こりうる。

:::message alert
何も指さないポインタを通してアクセスする
```cpp
int* p = nullptr;
std::println("{}", *p);
```
`p` は何も指していない。`*p` でアクセスした結果は定まらず、実行時に深刻な問題が起こりうる。
:::

ポインタはアドレスを直接扱えるぶん、できることが多い。本書では、他のオブジェクトを指すには参照を主に使い、ポインタは、何も指さない状態(`nullptr`)や、指す先を変える必要があるときに用いる。ポインタにまつわるそれ以上の扱いには、本書では立ち入らない。

## 指すということ

参照とポインタは、どちらも他のオブジェクトを指す。違いは、参照がオブジェクトではない別名であり、一度束縛したら変えられないのに対し、ポインタはアドレスを値として持つ一つのオブジェクトで、指す先を変えることも、何も指さない状態も表せる点にある。本書は、指すだけでよい場面では参照を、それ以上が要る場面でポインタを使う。

:::details 変数が持つのは値か、オブジェクトへの手がかりか
変数がオブジェクトそのものを表すのか、別の場所にあるオブジェクトへの手がかりを表すのかは、言語によって異なる。C++ では、変数は既定でオブジェクトそのものを表す。`int y = x;` は値をコピーして別のオブジェクトを作り、同じオブジェクトを別の名前で指したいときに、参照やポインタを明示して使う。一方、Java や Python では、クラスのインスタンスを入れた変数は、そのオブジェクトへの手がかりを表す。代入してもコピーは作られず、同じオブジェクトを指す。参照やポインタという型を書き分けることはない。Rust にも参照(`&`)があるが、どの参照がいつまで有効かが、ビルドの時点で検査される。
:::

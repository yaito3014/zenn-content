---
title: "基本型"
---

ここまでで、小さなプログラムを構成する要素を一通り見た。ここからは、それらを一つずつ詳しく扱っていく。最初は **型** である。

これまで本文で扱った型は、ほぼ `int` だけだった(条件分岐の章で `bool` を一度見た)。`int` や `bool`、`double`、`char` のような型は、まとめて **基本型** と呼ばれる。本章では、`int` 以外の基本型をいくつか見て、型ごとに何が違うのかを捉える。

整数の型の章で、型は値の種類を表し、種類ごとにその値にどのような操作ができるかが決まる、と述べた。型が違えば、扱える値も、その値にできる操作も違う。

## 真偽値の型 `bool`

条件分岐の章で、比較演算の結果は真偽値であり、その型は `bool` だと述べた。`bool` は、それ自体を変数の型としても使える。

```cpp
bool ok = true;
```

`bool` 型の値は `true` と `false` の2つだけである。`if` の条件に書けたのは、この `bool` 型の式であった。`bool` 型の値どうしを組み合わせる演算は、次章で扱う。

## 実数の型 `double`

`int` は整数しか表せない。小数を含む実数を扱うには `double` を使う。

```cpp
double pi = 3.14;
std::println("{}", pi);
```

これは `3.14` と出力する。`3.14` のように小数点を含むリテラルは、`double` 型の値になる。

整数の型のコラムで、`int` 同士の `/` は小数部を捨てた商になると述べた。`double` 同士の `/` は、小数を含む割り算になる。

```cpp
double half = 1.0 / 2.0;
```

`half` は `0.5` になる。`int` で `1 / 2` と書くと `0` になるのとは異なる。同じ `/` でも、値の型によって意味が変わる。

## 文字の型 `char`

一文字を表すには `char` を使う。

```cpp
char initial = 'a';
```

`'a'` のように、シングルクォートで囲んだ一文字のリテラルは、`char` 型の値になる。

一方、複数の文字を並べた文字列を1つの値として扱うには、`char` とは別に、標準ライブラリが提供する型を使う。文字列は続きの編で扱う。

## 表せる値には限りがある

型は、表せる値の範囲も定める。`int` がその例である。`int` で表せる整数には上限と下限があり、その境界がどこにあるかは処理系によって決まる。多くの環境では、`int` はおよそ -21 億から 21 億までを表せる。

この範囲を超える計算をすると、結果は定まらない。

:::message alert
`int` で表せる範囲を超える
```cpp
int big = 2000000000;
int sum = big + big;
std::println("{}", sum);
```
多くの環境で、`big + big`(およそ 40 億)は `int` で表せる範囲を超える。符号付き整数が範囲を超えたときの動作は規格で定められておらず、`sum` に何が入るかは保証されない。実行時に深刻な問題が起こりうる。
:::

## 型は、扱える値と操作を定める

ここで見たように、型ごとに、表せる値と、その値にできる操作が決まっている。`bool` は真偽値を、`double` は実数を、`char` は一文字を表し、`int` には表せる範囲がある。変数や式に型を与えることは、扱える値と操作を選ぶことにほかならない。

なお、基本型はこれらで尽きるわけではない。表せる範囲や精度の異なる整数・実数の型などもある。本書ではそれらに深く立ち入らず、整数には `int`、実数には `double` を主に使う。

次章からは、これらの型の値に対する演算を詳しく見ていく。

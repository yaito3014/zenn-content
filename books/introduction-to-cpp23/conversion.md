---
title: "型変換"
---

これまで、整数には `int`、実数には `double` と、型を混ぜずに使ってきた。だが、ある型の値を別の型に変えたい場面はある。値の型を別の型に変えることを **型変換** と呼ぶ。本章では、まず明示的に書く型変換を見て、次に、書かなくても起きる暗黙の型変換と、その制御を扱う。

## 明示的な変換

`double` の値を `int` にしたいとする。`static_cast` を使うと、型変換を明示的に書ける。

```cpp
double d = 3.9;
int n = static_cast<int>(d);
std::println("{}", n);
```

`static_cast<int>(d)` は、`double` の値 `d` を `int` に変換する式である。`double` から `int` への変換では小数部が捨てられ、`3.9` は `3` になる。出力は `3` である。`static_cast<変換先の型>(式)` と書く。

## 暗黙の変換

型変換は、`static_cast` を書かなくても起きることがある。これを **暗黙の変換** と呼ぶ。

```cpp
int n = 3.9;
std::println("{}", n);
```

`3.9` は `double` の値だが、これを `int` の変数 `n` に結びつけると、`int` への変換が起きる。`static_cast` を書いていないのに、`3.9` は `3` に変わる。出力は `3` である。

明示的な変換と違い、暗黙の変換はコードの上に現れない。値が意図せず変わっても、ビルドツールは何も指摘しないことが多い。ビルドが通ってしまうこの種の誤りこそ、本書がとくに注意するものである。

## 縮小変換と波括弧

`double` から `int` への変換のように、表せる値が狭くなる変換を **縮小変換** と呼ぶ。`3.9` を `int` にすると小数部が失われるのが、その例である。

波括弧 `{}` を使って初期化すると、縮小変換は許されなくなる。

:::message
波括弧での初期化で縮小変換する
```cpp
int n = {3.9};
```
`{}` での初期化は縮小変換を許さない。`3.9` を `int` にすると小数部が失われるため、ビルドに失敗する。
:::

波括弧で書いておけば、黙って値が変わる縮小変換を、ビルドの段階で捕まえられる。小数部を捨てることを承知のうえで変換するなら、`static_cast<int>(3.9)` と明示する。暗黙のうちに起こさず、意図を `static_cast` で表す。

## 変換コンストラクタと explicit

暗黙の変換は、自分で定義した型でも起きる。コンストラクタとデストラクタの章で書いた、`int` を受け取る `Counter(int start)` を考える。

```cpp
Counter c = 5;
```

`Counter c = 5;` と書くと、`int` の `5` から `Counter` への変換が暗黙に起きる。`Counter(int start)` が、`int` から `Counter` を作るコンストラクタだからである。このように暗黙の変換に使えるコンストラクタを **変換コンストラクタ** と呼ぶ。

意図しない暗黙の変換を防ぐには、コンストラクタに `explicit` を付ける。`Counter(int start)` の前に `explicit` と書く。

```cpp
explicit Counter(int start) : count(start)
{
}
```

:::message
explicit なコンストラクタで暗黙の変換をする
```cpp
Counter c = 5;
```
`Counter(int)` は `explicit` なので、`int` から `Counter` への暗黙の変換は禁じられる。ビルドに失敗する。
:::

`Counter` を作りたいなら、`Counter c(5);` と明示する。本書では、引数を1つ取るコンストラクタには `explicit` を付けることを基本とする。意図しない暗黙の変換を防げるからである。コンストラクタとデストラクタの章の `Counter(int)` にも、本来は `explicit` を付けるべきだった。

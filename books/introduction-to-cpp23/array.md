---
title: "配列"
---

同じ型の値を、いくつも並べてまとめて扱いたいことがある。それを **配列** で表す。本章では、配列の基本、範囲外アクセスという誤り、動的な配列、そして並んだ値を扱う標準ライブラリの道具への橋渡しを扱う。

## 配列

`int` の値を5個並べた配列は、次のように作る。

```cpp
int a[5] = {10, 20, 30, 40, 50};
```

`int a[5]` は、`int` を5個並べた配列を宣言する。`[5]` がその個数(大きさ)である。`= {...}` で各要素の初期値を与える。

要素には添字でアクセスする。添字は `0` から始まる。

```cpp
std::println("{}", a[0]);
std::println("{}", a[4]);
```

`a[0]` は先頭の要素 `10`、`a[4]` は5番目の要素 `50` である。大きさ5の配列の要素は、`a[0]` から `a[4]` までの5つである。

繰り返しの章で見た `for` で、添字を動かして各要素を順に扱える。

```cpp
for (int i = 0; i < 5; i = i + 1)
{
    std::println("{}", a[i]);
}
```

これは `10` から `50` までを順に出力する。

## 範囲外アクセス

配列の大きさは決まっている。その範囲の外の添字でアクセスしても、ビルドは止まらない。

:::message alert
配列の範囲外にアクセスする
```cpp
int a[5] = {10, 20, 30, 40, 50};
std::println("{}", a[10]);
```
`a` の要素は `a[0]` から `a[4]` までの5つしかない。`a[10]` はその範囲の外で、読み取った結果は定まらず、実行時に深刻な問題が起こりうる。
:::

範囲の内かどうかは、書く側が気をつけるしかない。配列自身が大きさを確かめてくれるわけではない。

## 動的な配列

ここまでの配列は、大きさを `[5]` のようにあらかじめ書いていた。大きさを実行時に決めたいときは、`new[]` で動的に配列を確保する。

```cpp
int n = 5;
int* p = new int[n];
p[0] = 10;
delete[] p;
```

`new int[n]` は、`int` を `n` 個並べた配列を動的に確保し、先頭の要素のアドレスを返す。`int*` の `p` で受け取り、`p[0]` のように添字でアクセスする。配列の名前 `a` も、`new[]` の戻り値 `p` も、どちらも先頭の要素を指す形で使える。破棄には、ひとつのオブジェクト用の `delete` ではなく、`delete[]` を使う。

動的な配列でも、確保した個数の範囲の外にアクセスすれば、やはり結果は定まらない。

## 並んだ値を扱う道具へ

生の配列は、大きさを自分で覚え、範囲を自分で守り、動的なものは `delete[]` を忘れずに書く必要がある。手動の管理は誤りを生みやすい。

並んだ値を扱う道具が標準ライブラリにある。`std::array`(大きさが固定)と `std::vector`(大きさを変えられる)は、確保・破棄・大きさの管理を引き受け、範囲の内かどうかを確かめる手立ても備える。生の配列を自分で扱う代わりに、こうした道具を使う。これらは標準ライブラリの機能で、続きの編で扱う。

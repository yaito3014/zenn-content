第1部の関数章では、`add` を例に、関数を「処理のまとまりに名前を与えたもの」と捉え、戻り値の型・名前・仮引数の並び・本体に分解した。本章では、その関数を型の観点から見直す。

先に、本書での捉え方を示しておく。関数は、受け取る引数の型と、返す戻り値の型によって特徴づけられる。`add` でいえば「`int` を2つ受け取り、`int` を1つ返す」関数である。関数を使う側は、本体の中身を知らなくても、これらの型さえ分かれば呼び出せる。

## 引数の型と戻り値の型

関数の章で分解した `add` を再掲する。

```cpp
int add(int a, int b)
{
    return a + b;
}
```

最初の `int` が戻り値の型、`(int a, int b)` が仮引数の並びで、その型はどちらも `int` である。すなわち `add` は「`int` を2つ受け取り、`int` を1つ返す」関数である。

`add` それ自体にも型がある。`add` の型は `int(int, int)` と書け、戻り値の型と仮引数の型からなる。名前 `add` は型には含まれない。

関数には、型とは別に **シグネチャ** と呼ばれるものもある。シグネチャには、関数の名前と仮引数の型の並びが含まれる。両者は次のように異なる。

- 関数の型 `int(int, int)`:戻り値の型と仮引数の型からなる。名前を含まない。
- シグネチャ:名前と仮引数の型を含む。戻り値の型は含まない。

シグネチャが戻り値の型を含まないことは、オーバーロードのところで効いてくる。

:::details シグネチャに何が含まれるか
シグネチャに何が含まれるかは、対象によって異なる。規格は、通常の関数・関数テンプレート・メンバ関数などについて、それぞれ個別に定めている。通常の関数では、名前と仮引数の型の並びのほかに、関数が属する名前空間(`std::println` の `std` のようなもの)も含む。関数テンプレートではさらに戻り値の型などが、メンバ関数では属するクラスや `const` などの修飾が加わる。本章で関わるのは名前と仮引数の型なので、本文ではそれらを挙げた。
:::

呼び出す側は、これらの型に従う。

```cpp
int result = add(2, 3);
```

`2` と `3` はどちらも `int` で、仮引数 `a`、`b` の型に合う。戻り値は `int` なので、`int` 型の `result` に結びつけられる。型の合わない実引数を渡すと、ビルドに失敗する。

:::message
型の合わない実引数を渡す
```cpp
add("2", "3");
```
`add` は `int` を2つ受け取るが、文字列リテラルは `int` に変換できない。ビルドに失敗する。
:::

## 値の渡し方

`add(2, 3)` のように呼び出すと、実引数の値が仮引数に結びつく。このとき仮引数は、実引数の値をコピーしたものになる。これを値渡しと呼ぶ。

仮引数はコピーなので、関数の中で仮引数を書き換えても、呼び出し側のオブジェクトは変わらない。

```cpp
void add_one(int n)
{
    n = n + 1;
}
```

```cpp
int x = 5;
add_one(x);
std::println("{}", x);
```

`add_one(x)` は、`x` の値 `5` を仮引数 `n` にコピーする。関数の中で `n` を `6` にしても、変わるのはコピーである `n` だけで、`x` は `5` のままである。出力は `5` になる。

値渡しでは、このように関数から呼び出し側のオブジェクトを変えられない。第1部の関数章で、関数は外側に変化を残すより値を返す形を主軸にすると述べた。`x` を増やした値が欲しいなら、結果を返して受け取る。

```cpp
int add_one(int n)
{
    return n + 1;
}
```

```cpp
int x = 5;
int y = add_one(x);
```

この形なら値渡しと整合する。一方、呼び出し側のオブジェクトそのものを指したい場合や、大きなオブジェクトのコピーを避けたい場合には、参照という仕組みを使う。これは次章で扱う。

## オーバーロード

同じ名前の関数を、引数の型を変えて複数宣言できる。これをオーバーロードと呼ぶ。

`int` を2つ受け取る `add` に加え、`double` を2つ受け取る `add` も宣言できる。

```cpp
int add(int a, int b)
{
    return a + b;
}

double add(double a, double b)
{
    return a + b;
}
```

名前はどちらも `add` だが、仮引数の型が異なる。呼び出したときにどちらが使われるかは、実引数の型で決まる。

```cpp
add(2, 3);      // int を2つ。int を返す add
add(2.5, 1.5);  // double を2つ。double を返す add
```

どのオーバーロードが選ばれるかは、実引数の型に基づいて決まる。その正確な規則は規格に細かく定められており、実引数の型が候補のどれとも完全には一致しない場合などには、規則に従って一つが選ばれるか、決められずにビルドに失敗する。本書では、型がはっきり一致する範囲で使う。

区別の手がかりはシグネチャである。`int` を2つ受け取る `add` と `double` を2つ受け取る `add` は、名前は同じだが仮引数の型が違うので、シグネチャが異なる。一方、戻り値の型だけが異なる関数は、名前も仮引数の型も同じでシグネチャが変わらず、オーバーロードとして区別されない。戻り値の型がシグネチャに含まれないことが、ここで効いている。

:::details リンクのときどう見分けるか
複数のオーバーロードは同じ名前を持つ。処理系は、関数の名前と仮引数の型をまとめて一意な記号名に変換する。これを **名前マングリング** と呼ぶ。リンクではこの記号名で関数を対応づけるので、同名のオーバーロードも別々の記号として区別される。記号名にはシグネチャに相当する情報が表れる。どんな記号名にするかは処理系(ABI)ごとの取り決めで、規格は定めていない。
:::

## デフォルト引数

仮引数に既定の値を与えておくと、その引数を省いて呼び出せる。この既定の値をデフォルト引数と呼ぶ。

```cpp
int next(int x, int step = 1)
{
    return x + step;
}
```

`step` に `= 1` と書いてある。

```cpp
next(5);     // step は 1。6 を返す
next(5, 3);  // step は 3。8 を返す
```

`next(5)` のように第2引数を省くと、`step` は既定値の `1` として扱われる。`next(5, 3)` のように渡せば、その値が使われる。

既定値は後ろの仮引数から順に与える。呼び出しで省けるのも後ろの引数からで、途中の引数だけを省くことはできない。

## 型から見た関数

本章では、関数を引数の型と戻り値の型から眺めてきた。名前と仮引数の型の並び(シグネチャ)が関数を見分ける手がかりとなり、オーバーロードはそれを使って同じ名前に複数の関数を持たせる。値渡しは、実引数が仮引数にどう結びつくかの既定の振る舞いである。デフォルト引数は、引数を省いたときの値を定める。いずれも、関数が何の型を受け取り何の型を返すかという観点から捉えられる。

---
title: "繰り返し"
---

代入の章の終わりで、書き換えが自然にあてはまる場面として繰り返しを挙げた。本章でその **繰り返し** を扱う。

先に、本書での繰り返しの捉え方を示しておく。繰り返しは、ある条件が成り立つ間、同じ文を何度も実行するしくみである。

## `while` の例

`1` から `5` までの整数を順に出力するコードを考える。これを `main` の本体に書いたとする。

```cpp
int i = 1;
while (i <= 5)
{
    std::println("{}", i);
    i = i + 1;
}
```

これを実行すると、`1` から `5` までが1行ずつ出力される。

## `while` 文

繰り返しの中心にあるのが、次の `while` 文である。

```cpp
while (i <= 5)
{
    std::println("{}", i);
    i = i + 1;
}
```

これは `while` 文と呼ばれる。`while` に続く括弧の中に条件を書き、その条件が成り立つ間、続く `{ }` の中の文が繰り返し実行される。条件 `i <= 5` は、条件分岐の章で見た比較演算による `bool` 型の式である。

このコードの動きを追う。

- はじめ `i` は `1` で、`i <= 5` が成り立つので本体が実行され、`1` が出力される。続く `i = i + 1;` で `i` は `2` になる。
- ふたたび条件が判定される。`i` は `2` で条件が成り立つので、`2` が出力され、`i` は `3` になる。
- これが繰り返され、`i` が `6` になると `i <= 5` が成り立たなくなり、繰り返しが終わる。

ここで `i = i + 1;` は、代入の章で見た代入である。繰り返しのたびに `i` の値を書き換えることで、出力する値が1つずつ進む。代入の章で「繰り返しでは代入が中心的な役割を担う」と述べたのは、この働きを指している。

もし `i = i + 1;` を書き忘れると、`i` は `1` のままで `i <= 5` がいつまでも成り立ち、繰り返しが終わらなくなる。繰り返しが終わるのは、値が変化して条件が成り立たなくなったときである。

## `for` 文

同じ繰り返しは、`for` 文でも書ける。

```cpp
for (int i = 1; i <= 5; i = i + 1)
{
    std::println("{}", i);
}
```

`for` に続く括弧の中には、繰り返しを制御する3つの部分が、この順に書かれている。

- `int i = 1;`：繰り返しを始める前に一度だけ行う、変数の宣言と初期化
- `i <= 5`：繰り返しのたびに判定する条件
- `i = i + 1`：本体を実行するたびに行う、変数の更新

`while` の例では、これら3つは別々の場所に散らばっていた。`for` 文は、繰り返しを制御するこの3つを括弧の中に1か所にまとめる。どちらで書いても、繰り返しの動きは同じである。

:::details for の括弧の中のセミコロン

本文では括弧の中を3つの部分に分けて説明したが、最初の `;` は、これらを区切る記号ではない。`int i = 1;` は変数の宣言であり、式と文の章で見た宣言文がそうであるように、それ自体が `;` で終わる。

`for` の括弧の中は、この `int i = 1;` という文に続けて、条件 `i <= 5`、`;`、更新の式 `i = i + 1` が並ぶ構造になっている。すなわち、括弧の中の2つの `;` のうち、1つ目は変数の宣言の末尾の `;` であり、条件と更新の式を区切っているのは2つ目の `;` だけである。

:::

## 宣言の並びとの関係

プログラムを宣言の並びとみる見方は、ここでも変わらない。`while` 文や `for` 文も、関数の本体に並ぶ文の1つである。繰り返しは、その文を何度実行するかを左右するものであって、宣言の並びという枠組みそのものを変えるわけではない。

## プログラムは宣言の並びである

次の C++ のプログラムを示す。これは整数の足し算を行い、その結果を出力する、動作する完全なプログラムである。

```cpp
#include <print>

int add(int a, int b)
{
    return a + b;
}

int main()
{
    int result = add(2, 3);
    std::println("{}", result);
}
```

このプログラムを実行すると、`5` と出力される。

ここで一つの問いを立てる。このプログラムは、何でできているのだろうか。

本書では、この問いに対して一つの見方を主軸に据える。すなわち、**このプログラムは宣言の並びである**、という見方である。

このコードには、宣言が三つ含まれている。`add`、`main`、そして `result` である。さしあたっては次のように捉えておけばよい。多くの宣言は、名前をプログラムに導入する。この三つの宣言も、いずれもそれぞれの名前をプログラムに導入している。

`add` は、関数の宣言である。`add` という名前をプログラムに導入し、それが「二つの整数を受け取って整数を返すもの」であることを述べている。

```cpp
int add(int a, int b)
{
    return a + b;
}
```

`main` も、関数の宣言である。ただし `main` は特別な関数であり、その特別さについては関数の章で改めて扱う。ここでは、`main` もまた関数の宣言の一つである、という点だけを確認しておけばよい。

```cpp
int main()
{
    int result = add(2, 3);
    std::println("{}", result);
}
```

そして `result` は、変数の宣言である。`result` という名前をプログラムに導入し、それが整数を保持するものであることを述べている。

`add`、`main`、`result` は、種類こそ関数であったり変数であったりと異なるが、いずれも名前をプログラムに導入する宣言である。**C++ のプログラムとは、このような宣言が並んだものである。** 本書は、この見方を軸に C++ を読み解いていく。

ここで「関数」「変数」という語を用いた。それぞれが具体的に何であるかは、後で述べる。この章では、どちらも宣言の一種であり、名前をプログラムに導入するという共通の働きを持つ、という点だけを押さえておけばよい。

なお、宣言が名前を導入するだけでなく、その名前が指す実体まで与えるとき、その宣言はとくに **定義** と呼ばれる。定義は宣言の一種である。上の三つの宣言は、いずれも実体まで与えているため定義でもある。この区別が実際に効いてくるのは、プログラムが複数のファイルにまたがるようになってからである。本書では、必要になった章で改めて扱う。さしあたっては、定義もまた宣言の一種である、という一点だけを覚えておけばよい。

このプログラムには、まだ説明していない要素がいくつか含まれている。`int` という型の名前、`return a + b` のような処理の記述、そして `std::println` による出力である。これらの詳しい意味は、後の章で順に扱う。さしあたっては、これらが「宣言の並び」という見方を損なうものではないことだけ述べておきたい。こうした処理の記述は、プログラムのどこにでも書けるわけではなく、つねに何らかの宣言の内側に現れる。

先頭の `#include <print>` だけは、少し性質が異なる。これは出力のための準備をプログラムに取り込む指示であり、これまで見てきた宣言とは別の種類のものである。本書ではこれをプリプロセッシングディレクティブと呼ぶが、その仕組みは後の章で扱う。さしあたっては、宣言の並びとしてのプログラムは、この準備の後に立ち現れると捉えておけばよい。

:::details C++ の文法における宣言

本書で「宣言」と呼んでいるものは、C++ の文法における `declaration` という構成要素に対応する。翻訳単位(おおまかにはソースファイル一つ分)は、文法上 `declaration` の並び(`declaration-seq`)として定義されており、この意味で「プログラムは宣言の並びである」という見方は、言語の文法構造そのものを言い表している。宣言の順序には意味があり、たとえば名前は原則として、それが使われるより前に宣言されていなければならない。

本文で「処理の記述」とぼかした部分は、文法上は `statement`(文)と呼ばれる。文はトップレベルには現れず、関数定義(これも `declaration` の一種である)の本体に書かれる。すなわち文は宣言と対等に並ぶものではなく、宣言の内側に包含される。

また、本文では多くの宣言が名前を導入すると述べたが、規格上はそうとは限らない。たとえばコンパイル時の条件を検査する `static_assert` 宣言のように、名前を導入しない宣言も存在する。「宣言とは名前を導入するものである」と言い切れない点は、頭の片隅に置いておくとよい。

他の言語では、文をプログラムの基本単位とみなす見方が一般的なこともある。プログラムをひとまとまりの手続きとして捉える見方も、それ自体は誤りではない。本書は、まず宣言という大きな枠組みがあり、文や処理はその内側にある、という階層に沿って C++ を捉える。

:::

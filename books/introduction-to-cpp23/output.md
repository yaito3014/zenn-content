---
title: "出力"
---

宣言の章では `std::println` を出力に使うものとして示し、関数の章では副作用を担う代表として、変数の章では `result` の値を画面に出力する例として、式と文の章では関数を呼ぶ式自体に外側への作用がある例として用いた。いずれも詳しい意味には立ち入ってこなかった。本章でこの `std::println` を扱う。

先に、本書での `std::println` の捉え方を示しておく。`std::println` は、値を文字列の形に整えて画面に出力する関数である。関数の章で副作用と呼んだ「関数の外側に変化を残す働き」の代表例にあたる。

## `std::println` の呼び出しを分解する

宣言の章で示したコードから、`std::println` を呼び出す式を取り出す。

```cpp
std::println("{}", result)
```

この呼び出しは次の要素からなる。

- `std::println`：関数の名前
- `"{}"`：書式文字列
- `result`：書式文字列に埋め込む値

この呼び出しでは、`std::println` は `"{}"` の中の `{}` という箇所を `result` の値(つまり `5`)に置き換えて、組み立てた文字列を画面に出力する。さらに末尾に改行を1つ加える。

## `{}` は値の埋め込み箇所を表す

`"{}"` のような文字列は **書式文字列** と呼ばれる。`{}` は、書式文字列の中で「ここに値を埋め込む」と指示する記法である。書式文字列の中の `{}` の数だけ、後ろに引数を渡す。

```cpp
int a = 2;
int b = 3;
std::println("{} と {} の和は {}", a, b, a + b);
```

これは画面に `2 と 3 の和は 5` と出力する。

`{}` の中に詳しい指定(桁数や進数など)を書くこともできる。これは続きの編で扱う。

## `println` の `ln` は改行

`std::println` は、組み立てた文字列の末尾に改行を1つ加えて出力する。名前の `ln` は line(行)を表し、出力を1行として扱うことに由来する。改行を加えない関数 `std::print` も用意されているが、本書では行ごとに区切る出力を主に使うため、`std::println` を中心に扱う。

## 値の型と出力

書式文字列の `{}` には、`int` 型の値だけでなく多くの型の値を埋め込める。`std::println` は、渡された値の型に応じた書き方で文字列を組み立てる。

たとえば、`{}` に渡した `result` の型は `int` であり、`int` 型の値は十進数の整数として書き出される。

## `std::println` を取り込む

宣言の章のコード冒頭にある `#include <print>` は、`std::println` をはじめとする出力の道具をプログラムから使えるようにする指示である。これを書かないと、`std::println` という名前が導入されないため、ビルドツールは未定義の名前として誤りを指摘する。

`#include` の仕組みそのものは次の章で扱う。`<print>` を取り込むと `std::println` が使えるようになる。

:::message
`#include <print>` を書き忘れる
```cpp
int main()
{
    std::println("{}", 5);
}
```
`std::println` という名前が導入されていないため、ビルドに失敗する。
:::

## 副作用としての出力

`std::println("{}", result)` は、呼び出した側に値を戻すために実行されるのではなく、画面に文字を表示するという外側の変化を起こすために実行される。これが、関数の章で副作用と呼んだ「関数の外側に変化を残す働き」にあたる。

関数の章で「副作用を主目的にしない」と述べたのは、`add` のように値から値を導く関数を組むときの姿勢であった。`std::println` のように副作用そのものを役目とする関数は、別に位置づける。両者を分けて使う形を本書では基本に据える。

## 宣言の並びとの関係

プログラムを宣言の並びとみる見方は、ここでも変わらない。`std::println` の呼び出しは、`main` の本体に書かれた1つの文(式文)であり、宣言の内部に現れる構成要素のひとつである。出力という働きの源は、宣言済みの `std::println` を呼び出すことにある。

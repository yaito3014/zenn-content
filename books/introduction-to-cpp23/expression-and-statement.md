---
title: "式と文"
---

宣言の章で「処理の記述(文と呼ばれる)」と紹介し、関数の章では `return 文` を、変数の章では初期化子の `=` の後ろに「式」が書かれると述べた。いずれも語だけ出して中身は伏せていた。本章でこれらを詳しく見る。

先に、本書での捉え方を示しておく。**式** は、評価すると値になるものである。**文** は、関数の本体のように、宣言の内側に並ぶものである。式と文は文法上別の区分だが、式は文の中に現れる。

## 式

宣言の章で示したコードには、次のような式が含まれている。

- `2`、`3`：リテラル
- `a`、`b`、`result`：名前を使う式
- `a + b`：演算子を使う式
- `add(2, 3)`：関数を呼び出す式

どれも、評価するとひとつの値になる。これが式に共通する性質である。

### リテラル

`2` や `3` のような、コードに直接書かれた値そのものをリテラルと呼ぶ。整数の値をそのまま書いたものは整数リテラルである。

### 名前を使う式

`a + b` の `a` や、`std::println("{}", result)` の `result` のように、すでに導入された名前を式として書ける。これを評価すると、その名前に結びついた値が取り出される。

### 演算子を使う式

`a + b` のように、演算子(`+`)と二つの式を組み合わせると、新しい式になる。評価すると、左右の式を評価した値どうしの演算結果が得られる。

### 関数を呼び出す式

`add(2, 3)` のように、関数の名前と括弧付きの実引数で関数を呼び出す式になる。評価すると、関数の本体が実行され、`return` で戻された値がこの式の値となる。`add(2, 3)` を評価すると `5` が得られる、というのは関数の章で見たとおりである。

### 式は型を持つ

すべての式は、評価したときに得られる値の型を持つ。`2` や `3` は `int` の値であり、`a + b` も `int` どうしの和なので `int` の値となる。`add(2, 3)` の値の型は、関数 `add` の戻り値の型と同じく `int` である。型の詳しい性質は別の章で改めて扱う。

## 文

関数の本体に並んでいるのは、文である。宣言の章で示したコードのうち、`main` の本体を再掲する。

```cpp
int main()
{
    int result = add(2, 3);
    std::println("{}", result);
}
```

ここには文が二つ並んでいる。

- `int result = add(2, 3);`：宣言を含む文
- `std::println("{}", result);`：式に `;` を付けた文

### 宣言文

`int result = add(2, 3);` は、変数の宣言を関数の本体に書いたものである。文として書かれた宣言を宣言文と呼ぶ。変数の章で扱った `result` の宣言が、これにあたる。

### 式文

`std::println("{}", result);` は、関数を呼び出す式 `std::println("{}", result)` の末尾に `;` を付けたものである。これを式文と呼ぶ。`std::println` の値は呼び出し側では使われない。それでも式文として書くのは、関数を呼び出すこと自体に外側への作用(画面への出力)があるためである。

### return 文

`add` の本体に並んでいた `return a + b;` は、関数の章で扱った `return 文` である。`return` の後ろに式を書き、その式を評価した値が、関数を呼び出した側に戻る。

### 文の末尾の `;`

ここまで見てきた文は、いずれも末尾に `;` を持つ。`;` を書き忘れると、ビルドツールはどこからどこまでが一つの文かを判断できず、誤りを指摘する。

:::message
文の末尾の `;` を書き忘れる
```cpp
int result = add(2, 3)
std::println("{}", result);
```
1 行目の末尾に `;` がないため、ビルドに失敗する。
:::

## 宣言の並びとの関係

プログラムが宣言の並びであるという見方は、ここでも変わらない。文と式は、宣言(関数の本体)の内側に現れる、より下の階層の構成要素である。
